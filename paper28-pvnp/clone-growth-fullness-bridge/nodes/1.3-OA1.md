## 1. Precise Statement of OA1

**Claim (OA1).** Let L be a Boolean constraint language (a finite
set of relations on {0,1}). Let f : {0,1}^k -> {0,1} be a k-ary
polymorphism of L (i.e., f preserves every relation R in L
coordinate-wise). Let M_Bool^L denote the sector of the Boolean
Bost-Connes factor corresponding to L. Then:

(a) f induces an automorphism alpha_f in Aut(M_Bool^L).

(b) If f is non-trivial (not a projection, not essentially unary),
    then alpha_f is outer: alpha_f not in Inn(M_Bool^L).

## 2. Construction of the Lift alpha_f

### 2.1 Setup

Let M_Bool^L be the type III_1 factor arising from the GNS
construction of the Boolean Bost-Connes system restricted to
language L. The factor acts on a Hilbert space H. Within
M_Bool^L, there exist "witness operators" W_R for each relation
R in L, encoding the solution space of R.

The key objects:
- For each relation R in L of arity m, the solution set
  Sol(R) = {(a_1,...,a_m) in {0,1}^m : R(a_1,...,a_m) = true}.
- The witness operator W_R, an element of M_Bool^L encoding Sol(R).

### 2.2 The natural construction

Given a k-ary polymorphism f : {0,1}^k -> {0,1} preserving L,
define alpha_f as follows.

**Step 1: Tensor amplification.**
Form the k-fold tensor product (M_Bool^L)^{otimes k}. This is
again a type III_1 factor. For each relation R in L of arity m,
we have k copies W_R^{(1)},...,W_R^{(k)} living in the
respective tensor factors.

**Step 2: Coordinatewise application of f.**
For each R in L and each coordinate position j in {1,...,m},
define the "f-combined" operator by applying f to the j-th
Boolean variable across all k copies. Concretely, if
W_R = sum_{a in Sol(R)} |a><a| (spectral decomposition over
the solution space), then:

  alpha_f(W_R) := sum_{a^{(1)},...,a^{(k)} in Sol(R)^k}
    |f(a^{(1)})><f(a^{(1)})| otimes ... (projected back)

More precisely, the polymorphism condition guarantees that if
a^{(1)},...,a^{(k)} in Sol(R), then
(f(a_1^{(1)},...,a_1^{(k)}), ..., f(a_m^{(1)},...,a_m^{(k)})) in Sol(R).
So alpha_f maps the k-fold solution space back into the 1-fold
solution space.

**Step 3: Compression back to M_Bool^L.**
The map alpha_f : (M_Bool^L)^{otimes k} -> M_Bool^L acts by
f-combination on witness operators and extends to all of M_Bool^L
by the universal property of the factor.

### 2.3 The endomorphism problem

**WARNING:** The construction above is NOT an automorphism of
M_Bool^L. It is a map FROM (M_Bool^L)^{otimes k} TO M_Bool^L --
a k-ary operation on the algebra, not a unary one. To get an
automorphism of M_Bool^L, we need a different construction.

### 2.4 Revised construction: diagonal embedding + polymorphism

**Step 1: Diagonal embedding.**
Embed M_Bool^L diagonally into (M_Bool^L)^{otimes k} via
Delta(x) = x otimes x otimes ... otimes x (k copies).

**Step 2: Define alpha_f on generators.**
For each witness operator W_R in M_Bool^L, define:

  alpha_f(W_R) := Phi_f(W_R otimes ... otimes W_R)

where Phi_f is the f-combination map from Step 2 above. The
polymorphism condition ensures alpha_f(W_R) is again in M_Bool^L
(it encodes solutions of R combined via f).

**Step 3: Verify automorphism properties.**
- Well-defined: by the polymorphism condition.
- Multiplicative: requires f to be compatible with the algebraic
  structure of M_Bool^L. This is where the detailed verification
  is needed.
- Invertible: requires that f has a "polymorphism inverse" in a
  suitable sense. For idempotent operations (f(x,...,x) = x),
  alpha_f fixes the diagonal, so it has the right to be invertible
  on M_Bool^L.

### 2.5 Alternative construction: Bogoljubov-type automorphism

There is a cleaner construction using the Fock space structure
of M_Bool^L (if it has one). If M_Bool^L is generated by
creation/annihilation operators a_j, a_j* indexed by Boolean
variables, then a polymorphism f : {0,1}^k -> {0,1} can be
lifted to a Bogoljubov automorphism:

  alpha_f(a_j) = f(a_j^{(1)}, ..., a_j^{(k)})

where the a_j^{(i)} are k independent copies. This is the
"second quantization" of the polymorphism. Bogoljubov
automorphisms of free Araki-Woods factors are known to have
well-understood outerness properties (see Houdayer's work).

**GAP:** This construction requires M_Bool^L to have an
Araki-Woods or free Araki-Woods realization. Whether the Boolean
Bost-Connes construction produces such a realization is not
established.

## 3. The Outerness Argument -- and the Critical Obstruction

### 3.1 The naive argument (from the REFRAME)

The REFRAME suggested: sigma_t (the modular automorphism group)
is 1-ary (scales by a phase), while alpha_f is genuinely k-ary
(mixes k copies). Therefore alpha_f is not in the closure of
{sigma_t}, hence outer.

**This argument has a critical gap.** It conflates the
construction space (k copies) with the automorphism target
(M_Bool^L itself). After compression to M_Bool^L, the
automorphism alpha_f is a map M_Bool^L -> M_Bool^L, and its
"k-ary origin" is no longer visible at the level of the single
factor. An automorphism of a factor is either inner or outer
based on its action on that single factor, not on how it was
constructed.

### 3.2 The injective III_1 obstruction (CRITICAL FINDING)

The following theorem is the central obstruction to OA1:

**Theorem (Kawahigashi-Sutherland-Takesaki 1992, refining
Connes).** Every automorphism of the AFD (approximately finite
dimensional = injective = hyperfinite) type III_1 factor R_infty
is approximately inner.

**Theorem (Connes 1975-1977).** For the injective type III_1
factor R_infty:
- The modular automorphism group sigma_t gives an embedding
  R -> Out(R_infty).
- An automorphism alpha is inner if and only if
  mod(alpha) = 0 in R/Ker(mod).
- For type III_1: Ker(mod) = {0} (the Connes S-invariant is
  all of R_+), so the modular homomorphism
  R -> Out(R_infty) is injective.
- However, Out(R_infty) is MUCH LARGER than just the image
  of the modular flow. The outer automorphism group of the
  injective III_1 factor is enormous (it contains all
  approximately inner automorphisms modulo inner ones, and
  much more).

**The critical question is: Is M_Bool^L injective?**

If M_Bool^L is injective (= hyperfinite), then:
- M_Bool^L is isomorphic to the unique injective type III_1
  factor R_infty (by Connes' uniqueness theorem + Haagerup).
- Out(R_infty) is huge -- it contains continuous families
  parameterized by R (the modular flow) and more.
- The polymorphism lift alpha_f would automatically be
  approximately inner (KST theorem).
- But approximately inner does NOT mean inner. The distinction
  between approximately inner and inner is precisely the
  content of Out(M).
- So outerness is NOT excluded -- but neither is innerness
  guaranteed.

If M_Bool^L is NOT injective (which is more likely for an
interesting construction):
- M_Bool^L is a non-injective type III_1 factor.
- Its Out group can range from as small as R (Chakraborty 2023,
  arXiv:2312.04702, using amalgamated free products) to very
  large.
- The structure of Out(M) for non-injective factors is much
  less understood.

### 3.3 The fullness dichotomy (Marrakchi, Houdayer-Marrakchi)

**Theorem (Marrakchi, arXiv:1605.09613, Crelle 2019).**
A type III_1 factor M is full (= no non-trivial centralizing
sequences) if and only if the modular automorphism group has
a spectral gap.

**Theorem (Marrakchi-Vaes, arXiv:2201.01055, 2022).**
If M is a full factor and alpha : G -> Aut(M) is an outer
action of a locally compact group G whose image in Out(M)
is closed, then:
(a) The action is automatically strictly outer (the relative
    commutant M' cap (M rtimes G) is trivial).
(b) The crossed product M rtimes G is also full.

**Relevance to OA1:** These results say that IF alpha_f is
outer AND the clone group's image in Out(M_Bool^L) is closed,
THEN the crossed product preserves fullness. But they do not
help prove that alpha_f is outer in the first place.

For the fullness direction relevant to the bridge:
- Full factor <=> spectral gap <=> Inn(M) is closed in Aut(M)
  (in the u-topology).
- Non-full factor <=> no spectral gap <=> Inn(M) is NOT closed
  <=> there exist approximately inner automorphisms that are
  outer.

**Key insight:** In a NON-FULL factor, Inn(M) is not closed,
so there are outer automorphisms that are limits of inner
automorphisms. This means Out(M) has a non-trivial topology --
it is NOT discrete. The non-discreteness of Out(M) IS the
non-fullness.

### 3.4 The structural picture

For the bridge theorem to work, we need:

| Language type | M_Bool^L property | Out(M_Bool^L) | Clone growth |
|---|---|---|---|
| Taylor (P-time) | Non-full | Non-discrete (continuous part) | Exponential |
| Non-Taylor (NPC) | Full | Discrete (only modular flow R) | Linear |

The bridge identifies:
- Exponential clone growth <-> non-discrete Out <-> non-full
- Linear clone growth <-> discrete Out <-> full

For this to work, we need:
1. Each non-trivial polymorphism contributes a DISTINCT element
   of Out(M_Bool^L).
2. For Taylor languages, the exponentially many polymorphisms
   generate a non-discrete subset of Out(M_Bool^L).
3. For non-Taylor languages, the linearly many (essentially
   unary) polymorphisms generate only a discrete subset.

### 3.5 Why outerness should hold (modular obstruction argument)

Here is the strongest argument I can construct for outerness,
avoiding the killed approaches:

**Argument.** Let f be a non-trivial k-ary polymorphism of L.
Consider the modular invariant mod : Aut(M_Bool^L) -> R.

The modular automorphism sigma_t has mod(sigma_t) = t.
An inner automorphism Ad(u) has mod(Ad(u)) = 0.

The polymorphism automorphism alpha_f acts on M_Bool^L by
mixing k copies of the Boolean structure. Compute its modular
invariant:

  mod(alpha_f) = ?

If mod(alpha_f) != 0, then alpha_f is outer (since inner
automorphisms have trivial mod).

**The computation:** Under the Boolean BC dynamics, the modular
flow is sigma_t(mu_C) = (depth C)^{it} mu_C. The polymorphism
f combines k circuits into one. If f has "depth k" in some
sense (the f-combination increases circuit depth), then
mod(alpha_f) = log(k) or similar. But this requires a precise
computation that depends on the structure of M_Bool^L.

**GAP:** The value of mod(alpha_f) depends on how alpha_f
interacts with the modular flow, which depends on the precise
construction of M_Bool^L. This is not determined by the
abstract setup alone.

### 3.6 Alternative argument: non-fullness implies outer automorphisms exist

**Theorem.** If M_Bool^L is non-full (which we claim for
Taylor languages), then Inn(M_Bool^L) is not closed in
Aut(M_Bool^L), which means there exist outer automorphisms
that are approximately inner.

**The question becomes:** Are the polymorphism automorphisms
among these outer-but-approximately-inner automorphisms?

This reverses the logic of OA1. Instead of:
  "polymorphisms -> outer automorphisms -> non-full"
we consider:
  "non-full -> outer automorphisms exist -> are they polymorphisms?"

This is closer to the backward direction of the bridge
(non-full -> Taylor) which is node 1.4's territory. But it
suggests a different proof strategy for OA1: instead of proving
outerness directly, prove that the polymorphism action is
precisely the mechanism by which Inn(M_Bool^L) fails to be
closed.

## 4. Literature Found and How It Applies

### 4.1 Connes (1975-1977): Classification of automorphisms of injective factors

- **Source:** Connes, "Outer conjugacy classes of automorphisms
  of factors," Ann. Sci. ENS 8(3):383-419, 1975.
  Connes, "Classification of injective factors," Ann. Math. 104:73-115, 1976.
- **Key result:** For injective type III_1, Out(R_infty) contains
  the modular flow R as a subgroup. The classification of
  automorphisms up to outer conjugacy uses mod and an additional
  Connes invariant.
- **Applies to OA1:** If M_Bool^L is injective, Out is completely
  classified and we can check whether alpha_f is inner by
  computing mod(alpha_f).

### 4.2 Kawahigashi-Sutherland-Takesaki (1992): Structure of Aut for injective III_1

- **Source:** Acta Math. 169:105-130.
- **Key result:** Every automorphism of the AFD type III_1 factor
  is approximately inner. Classification up to cocycle conjugacy
  for discrete abelian group actions.
- **Applies to OA1:** If M_Bool^L is injective, alpha_f is
  automatically approximately inner. Outerness is then
  equivalent to alpha_f not being inner, which is determined by
  the Connes invariants.

### 4.3 Marrakchi (2016/2019): Spectral gap characterization of full type III factors

- **Source:** arXiv:1605.09613, Crelle 2019.
- **Key result:** Type III_1 factor M is full iff modular
  automorphism group has spectral gap. Full => Inn(M) closed =>
  Out(M) = Aut(M)/Inn(M) has discrete topology.
- **Applies to OA1:** Provides the target: we need non-fullness
  (no spectral gap) for Taylor languages and fullness (spectral
  gap) for non-Taylor languages. The spectral gap is the
  operator-algebraic invariant that corresponds to clone growth.

### 4.4 Marrakchi-Vaes (2022): Spectral gap and strict outerness

- **Source:** arXiv:2201.01055.
- **Key result:** Outer actions of locally compact groups on full
  factors are automatically strictly outer. If the Out image is
  closed, the crossed product is full.
- **Applies to OA1:** If we can establish outerness, this
  theorem gives strict outerness for free. But it does not help
  prove outerness itself.

### 4.5 Chakraborty (2023): Smallest outer automorphism group

- **Source:** arXiv:2312.04702.
- **Key result:** There exists a full type III_1 factor M with
  Out(M) = R (just the modular flow, nothing else). This is the
  smallest possible Out for type III_1.
- **Applies to OA1:** Shows that full factors CAN have minimal
  Out. For the bridge theorem, this is consistent: NPC languages
  (full factors) should have small Out (just R from the modular
  flow), while P-time languages (non-full factors) should have
  large Out (R plus polymorphism contributions).

### 4.6 Haagerup (1987): Connes bicentralizer problem

- **Source:** Haagerup, "Connes' bicentralizer problem and
  uniqueness of the injective factor of type III_1,"
  Acta Math. 158:95-148, 1987.
- **Key result:** The injective type III_1 factor is unique
  (answering Connes' question). The bicentralizer of any
  faithful normal state on the injective III_1 factor is trivial.
- **Applies to OA1:** The trivial bicentralizer means the
  injective III_1 factor has very rigid internal structure,
  constraining what can be inner.

### 4.7 No prior work connecting polymorphisms to operator algebra automorphisms

The search for "polymorphism action operator algebra" and
"clone action von Neumann algebra" returned no results connecting
universal algebra polymorphisms to operator algebra automorphisms.
This confirms that OA1, if correct, would be genuinely new
mathematics -- a novel bridge between universal algebra and
operator algebra.

## 5. Self-Suspicion Section

### 5.1 Suspicion 1: M_Bool^L might be injective, killing the bridge

**The threat:** If M_Bool^L is the injective III_1 factor
R_infty for ALL languages L, then M_Bool^L is the SAME factor
for P-time and NPC languages. The factor cannot distinguish them.
Fullness vs non-fullness would be meaningless because R_infty
is non-full (it is approximately finite dimensional, hence has
non-trivial centralizing sequences, hence is non-full).

**Assessment:** This would be fatal. The bridge requires
different languages to produce DIFFERENT factors (or at least
factors with different fullness properties). If all M_Bool^L
are isomorphic to R_infty, the bridge collapses.

**Backward-verification check:** Is R_infty full or non-full?
R_infty is NOT full -- it has centralizing sequences (it is
AFD). So if all M_Bool^L = R_infty, then ALL sectors are
non-full, and fullness never distinguishes anything.

**Mitigation:** The construction of M_Bool^L in the programme
(preprint section 3) uses circuit depth as the modular parameter.
Different languages L restrict which circuits are available,
potentially producing non-isomorphic factors. In particular,
NPC languages might produce non-injective factors (because the
computational structure is too complex for AFD).

### 5.2 Suspicion 2: The polymorphism lift might not be well-defined as an automorphism

**The threat:** The construction in section 2 involves
compressing a k-ary operation to a unary one. This compression
might not preserve the algebraic structure of M_Bool^L. The map
alpha_f might be an endomorphism but not an automorphism (not
surjective), or it might not even be multiplicative.

**Assessment:** This is a genuine gap. The construction is
sketched but not verified. The multiplicativity of alpha_f
depends on how the polymorphism interacts with the product
structure of M_Bool^L, which is not obvious.

**Backward-verification check:** For a concrete case (e.g.,
f = majority for 2-SAT), does the construction produce a
well-defined automorphism? This should be checkable in a finite
approximation.

### 5.3 Suspicion 3: Outerness might hold but be vacuous for the bridge

**The threat:** Even if alpha_f is outer for all non-trivial
polymorphisms, the map f -> [alpha_f] in Out(M_Bool^L) might
have enormous kernel -- many distinct polymorphisms could map to
the same outer class. If the image is always finite (or always
just R), then exponential clone growth does not produce a
non-discrete Out image, and the bridge still collapses.

**Assessment:** This is the injectivity question (node 1.4,
OA2). Even if OA1 succeeds, the bridge needs OA2: distinct
polymorphisms -> distinct outer classes. If the kernel of the
lift is large, the growth rate is not preserved.

**Backward-verification check:** For 2-SAT with the majority
polymorphism, how many distinct outer classes arise from the
arity-k clone? The clone grows exponentially; does the Out
image also grow?

## 6. Decomposition (since status is BLOCKED-DECOMPOSED)

### 6.1 The blocking obstruction

OA1 is blocked by the following unresolved structural question:

**Q_struct:** Is M_Bool^L injective for all L, or does it
depend on L?

If injective for all L: the bridge likely collapses
(Suspicion 5.1). The proof strategy must be fundamentally
revised.

If L-dependent: the proof strategy is viable, but requires
establishing:
- M_Bool^L is non-injective (hence non-full possible) for
  NPC languages.
- M_Bool^L is injective (or at least non-full) for P-time
  languages.

### 6.2 Sub-nodes

**Node 1.3.1: Q_struct -- Injectivity of M_Bool^L.**
Determine whether M_Bool^L is injective. This requires
understanding the construction in preprint section 3
(KEY LEMMA 3.4.3). If M_Bool^L is a group measure space
construction L^infty(X) rtimes G, then injectivity depends
on the amenability of G.

Candidate resolution: The Bost-Connes system uses the semigroup
N* (multiplicative positive integers). The group generated by
N* is Q_+* (positive rationals), which is abelian, hence
amenable. This suggests M_Bool (the full factor) might be
injective. BUT: the Boolean restriction to language L might
introduce non-amenable structure (e.g., through the symmetry
group of the constraint language).

**Node 1.3.2: Construction verification.**
For a specific Taylor language (e.g., 2-SAT with majority),
explicitly construct alpha_f and verify it is:
(a) well-defined,
(b) multiplicative,
(c) an automorphism (invertible).

**Node 1.3.3: Outerness via modular invariant.**
If M_Bool^L is injective: compute mod(alpha_f) and check
whether it is nonzero. If mod(alpha_f) != 0, alpha_f is outer.
If mod(alpha_f) = 0 but alpha_f is not inner, use the finer
Connes invariants (centrally trivial vs. approximately inner
classification).

If M_Bool^L is non-injective: a different outerness argument
is needed. The modular invariant still provides a necessary
condition for innerness, but the KST classification does not
apply.

**Node 1.3.4: Alternative bridge architecture.**
If Suspicion 5.1 materializes (M_Bool^L is injective for all L),
the bridge theorem needs restructuring. Possible alternative:
instead of using Out(M_Bool^L) to distinguish languages, use
the INCLUSION structure M_Bool^L subset M_Bool, and show that
the polymorphism action on the relative commutant or the Jones
index distinguishes P-time from NPC.

### 6.3 Priority ordering

1. **Node 1.3.1** (highest priority): resolve Q_struct. This
   determines which branch of the proof to pursue.
2. **Node 1.3.2** (second): construct alpha_f explicitly for
   a concrete case.
3. **Node 1.3.3** (third): prove outerness using the
   appropriate tools (modular invariant or alternative).
4. **Node 1.3.4** (contingency): activated only if 1.3.1
   shows M_Bool^L is injective for all L.

## 7. Summary Assessment

### What was accomplished

1. The lift alpha_f was constructed in two forms (sections 2.2-2.5),
   but neither is verified to produce a well-defined automorphism.

2. A critical structural obstruction was identified: the
   Kawahigashi-Sutherland-Takesaki theorem that every automorphism
   of the injective III_1 factor is approximately inner, combined
   with the non-fullness of the injective III_1 factor, means that
   if M_Bool^L is injective, the fullness-based bridge CANNOT work
   (all sectors would be non-full).

3. The programme's bridge theorem requires M_Bool^L to be
   NON-injective for NPC languages (so that fullness is possible)
   and possibly injective or at least non-full for P-time languages.
   This is a new structural requirement that was not explicit in
   the blackboard.

4. No prior work connecting polymorphisms to operator algebra
   automorphisms was found, confirming novelty but also confirming
   that the construction has no template to follow.

### What remains

The bottleneck has shifted from "is alpha_f outer?" to the
more fundamental "what is the type of M_Bool^L?" The outerness
question only becomes meaningful once we know whether M_Bool^L
is injective.

## 8. Proposed New Approach (for Runner consideration)

### 8.1 The fullness-first strategy

Instead of proving OA1 directly (polymorphism -> outer
automorphism -> non-full), consider proving the bridge
via a fullness-first approach:

1. **Directly characterize fullness of M_Bool^L** in terms of
   properties of L, without going through Out(M_Bool^L).

2. **Use the Marrakchi spectral gap criterion** (arXiv:1605.09613):
   M_Bool^L is full iff the modular flow has a spectral gap.
   The spectral gap can potentially be computed directly from
   the constraint structure of L.

3. **Connect spectral gap to Taylor property** without
   polymorphism lifts: show that the spectral gap of the
   modular flow on M_Bool^L is zero iff L has a Taylor
   polymorphism.

This bypasses Out(M_Bool^L) entirely and connects fullness
directly to the Taylor property via the spectral gap.

**Advantage:** Avoids the need for OA1 and OA2 entirely.
**Risk:** The spectral gap computation requires detailed
knowledge of M_Bool^L's modular flow, which brings us back
to the same structural questions.

### 8.2 The subfactor strategy

Instead of using fullness of a single factor, use the
INCLUSION M_Bool^L subset M_Bool:

1. Compute the Jones index [M_Bool : M_Bool^L].
2. Show that the Jones index distinguishes Taylor from
   non-Taylor languages.
3. Use the Jones index theory to derive P != NP.

This is closer to the original preprint's section 4
approach (Jones inclusion M_Bool^P subset M_Bool^NP).
