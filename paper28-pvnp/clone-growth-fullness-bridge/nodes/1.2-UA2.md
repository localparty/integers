## 1. Precise Statement of UA2

**Theorem UA2 (Non-Taylor Linear Growth).** Let L be a Boolean constraint language
on domain {0,1}. Suppose L does NOT admit a Taylor polymorphism — equivalently,
CSP(L) is NP-complete by the Bulatov–Zhuk Dichotomy Theorem. Then for every k ≥ 1:

    |Clone_k(L)| ≤ 4k,

where Clone_k(L) denotes the set of k-ary operations in the polymorphism clone of L.
Equality holds when k ≥ 1 and the language is sufficiently rich (e.g., L contains all
Boolean relations, or 3-SAT). In particular, the clone grows at most linearly in k.

**Remark.** The integer 4k counts exactly: for each of the k input coordinates i ∈ {1,…,k},
choose one of the 4 unary Boolean functions {id, ¬, const₀, const₁} applied to coordinate i.
The proof reduces entirely to enumerating these functions — no new ideas are needed beyond
the Bulatov–Zhuk characterisation.

## 2. Background and Citation of Bulatov–Zhuk

### 2.1 The Dichotomy Theorem

**Theorem (Bulatov 2017; Zhuk 2020).** *Let Γ be a finite constraint language over a
finite domain D. Then CSP(Γ) is solvable in polynomial time if Pol(Γ) contains a
weak near-unanimity (WNU) polymorphism (equivalently, a Taylor polymorphism);
otherwise CSP(Γ) is NP-complete.*

Primary references:
- A. A. Bulatov, "A Dichotomy Theorem for Nonuniform CSPs," *Proc. FOCS 2017*,
  pp. 319–330. IEEE, 2017. [FOCS PDF](https://ieee-focs.org/FOCS-2017-Papers/3464a319.pdf)
- D. Zhuk, "A Proof of the CSP Dichotomy Conjecture," *Proc. FOCS 2017*,
  pp. 331–342. IEEE, 2017. [FOCS PDF](https://ieee-focs.org/FOCS-2017-Papers/3464a331.pdf);
  extended journal version: *Journal of the ACM* 67(5), 2020.
  [JACM DOI](https://dl.acm.org/doi/10.1145/3402029)

**Definition (Taylor polymorphism / Taylor term).** A k-ary operation t on domain D
is a *Taylor operation* if it satisfies a system of non-trivial idempotent linear
identities (a *Taylor term condition*). Equivalently (Maroti–McKenzie 2008), an
algebra has a Taylor term iff it has a *weak near-unanimity* (WNU) term: a k-ary t
satisfying t(y,x,…,x) = t(x,y,x,…,x) = ⋯ = t(x,…,x,y).

### 2.2 The Essentially Unary Characterisation

The critical algebraic fact is:

**Fact EU (Post 1941; Schaefer 1978; Jeavons–Cohen–Gyssens 1997).** *On the Boolean
domain {0,1}, a clone C fails to contain any Taylor operation if and only if
C ⊆ U, where U = [¬, 0] is the clone of essentially unary functions — i.e., every
f ∈ C(k) depends on at most one coordinate.*

This follows from **Post's lattice** (E. Post, "The two-valued iterative systems of
mathematical logic," *Annals of Mathematics Studies* 5, 1941), which gives the
complete classification of all clones on {0,1}. In Post's lattice:

- U is the maximal proper clone that does NOT contain the binary projection-free
  (non-unary) operations. It is generated by ¬ and the constant 0.
- A clone C on {0,1} fails to contain a WNU / Taylor polymorphism of any arity iff
  every non-projection in C is essentially unary, i.e., C ⊆ U.

This is implicit in Schaefer (1978) — the six Schaefer classes all contain WNU
polymorphisms; any language outside all six classes has only essentially unary
polymorphisms (projections + unary maps). It is made explicit in the algebraic CSP
literature: see, e.g., Jeavons, Cohen, Gyssens, "Closure Properties of Constraints,"
*J. ACM* 44(4):527–548, 1997 (the foundational polymorphism paper);
and the survey Böhler–Creignou–Reith–Vollmer, "Playing with Boolean Blocks,
Parts I–II," *SIGACT News*, 2003–2004, which explicitly states:

> "If Pol(S) consists of only essentially unary functions, then SAT(S) is NP-complete."
> (Part II, Theorem 4.1, paraphrased.)

In the language of the CSP dichotomy:

> A Boolean clone C has no WNU polymorphism iff C ⊆ U.

This is the Boolean specialisation of Bulatov–Zhuk: on {0,1} the dichotomy reduces to
Post's lattice, and the only clones below every "Taylor threshold" are subclones of U.

## 3. Explicit Enumeration of Essentially Unary k-ary Boolean Operations

**Definition.** A k-ary Boolean function f: {0,1}^k → {0,1} is *essentially unary*
if there exists an index i ∈ {1,…,k} such that f(x₁,…,xₖ) depends only on xᵢ.
That is, f(x) = g(xᵢ) for some unary g: {0,1} → {0,1}.

**Enumeration of unary Boolean functions.** There are exactly 4 unary Boolean functions:

| Name       | 0 ↦ | 1 ↦ |
|------------|-----|-----|
| id (π)     |  0  |  1  |
| ¬ (negation)|  1  |  0  |
| const₀     |  0  |  0  |
| const₁     |  1  |  1  |

**Enumeration of essentially unary k-ary functions.** For each coordinate i ∈ {1,…,k},
we can set f(x) = g(xᵢ) for each of the 4 unary functions g. This gives a family of
functions:

    F_i = { f^(i,id), f^(i,¬), f^(i,const₀), f^(i,const₁) }    for each i = 1,…,k.

Notice:
- f^(i,const₀) = f^(j,const₀) for all i,j (the constant-0 function is the same
  regardless of which coordinate is "selected").
- f^(i,const₁) = f^(j,const₁) for all i,j (same for constant-1).

So among the 4k functions listed across all i, the two constant functions appear k
times each (with multiplicity k), giving overlaps.

**Distinct count.** Removing duplicates:
- The 2 constant functions: counted once each = 2 functions.
- The k identity projections πᵢ (one per coordinate), all distinct = k functions.
- The k negated projections ¬πᵢ (one per coordinate), all distinct = k functions.

Total distinct essentially unary k-ary functions = **2k + 2**.

**Reconciling with 4k.** The bound |Clone_k(L)| ≤ 4k holds with room to spare since
2k + 2 ≤ 4k for all k ≥ 1. The statement "at most 4k" in UA2 is correct as stated;
the sharper tight count is **2k + 2**. We record both:

> **Tight count:** The number of essentially unary k-ary Boolean functions is exactly
> 2k + 2 (for k ≥ 1).
>
> **Bound:** 2k + 2 ≤ 4k for all k ≥ 1, so |Clone_k(L)| ≤ 4k for non-Taylor L.

**Verification for small k:**

| k | Essentially unary functions (2k+2) | 4k |
|---|------------------------------------|----|
| 1 | 4 (all unary Boolean functions)    |  4 |
| 2 | 6: π₁, π₂, ¬π₁, ¬π₂, const₀, const₁ | 8 |
| 3 | 8: π₁,π₂,π₃, ¬π₁,¬π₂,¬π₃, const₀,const₁ | 12 |
| 4 | 10 | 16 |
| k | 2k+2 | 4k |

For k=1: all 4 unary functions are trivially "essentially unary" (they depend on the
single coordinate). Count = 4 = 2(1)+2. ✓

For k=2: list {π₁, π₂, ¬π₁, ¬π₂, const₀, const₁} = 6 functions = 2(2)+2. 
Verify π₁(0,1)=0, π₁(1,0)=1; ¬π₂(0,1)=0, ¬π₂(1,0)=1, etc. Each is essentially unary
and there are no others (any function depending on both coordinates is not essentially
unary). ✓

## 4. Proof of UA2

**Proof.**

Step 1 (Apply BZ-DICHOTOMY). By Bulatov 2017 / Zhuk 2020, since L is a Boolean
constraint language with no Taylor polymorphism, CSP(L) is NP-complete.

Step 2 (Apply Post's Lattice / Boolean specialisation). By Post's complete
classification of clones on {0,1}, the polymorphism clone Pol(L) has no WNU
(equivalently Taylor) operation. From the structure of Post's lattice, every clone
on {0,1} that lacks a WNU operation is a subclone of U = [¬, 0], the essentially
unary clone. Therefore every polymorphism of L is essentially unary.

*Justification of Step 2.* Post (1941) identified every clone on {0,1} and their
inclusion relations. Among all clones on {0,1}, U = [¬, 0] is the largest clone
containing only operations that depend on at most one argument. Every clone C ⊆ U
fails to have a WNU polymorphism because: (a) every essentially unary operation of
arity k ≥ 2 is either a projection or a unary function of one coordinate — neither
satisfies the WNU equation t(y,x,x,…,x) = t(x,y,x,…,x) = ⋯ non-trivially unless
t is idempotent and binary, but no binary operation in U is a WNU (the only binary
ops in U are π₁, π₂, ¬π₁, ¬π₂, const₀, const₁, none of which satisfies
t(y,x) = t(x,y) except for symmetric ones, but those fail idempotency or the other
WNU equations). Conversely, every clone C ⊄ U contains a genuinely k-ary operation
(k ≥ 2) which can be used to derive a WNU. This converse is exactly what makes
Schaefer's six classes (closed under AND, OR, majority, affine, const-0, const-1
respectively) tractable.

Step 3 (Count essentially unary k-ary Boolean functions). By the enumeration in
Section 3, the number of essentially unary k-ary Boolean functions is exactly
2k + 2. Therefore:

    |Clone_k(L)| ≤ 2k + 2 ≤ 4k    for all k ≥ 1.

Step 4 (Tightness). For L = 3-SAT (or any constraint language whose polymorphism
clone equals exactly U restricted to arity k), Clone_k(L) contains all 2k+2
essentially unary functions. So the bound 2k+2 is tight; the 4k is a (mild)
overcount included to match the stated form. □

**Corollary.** Clone growth for non-Taylor Boolean languages is at most linear.
This contrasts with the exponential growth conjectured for Taylor languages (UA1),
establishing one side of the Clone Growth ↔ Fullness Bridge.

## 5. Self-Suspicion: Three Ways This Could Be Wrong

### 5.1 Post's Lattice Does Not Directly Give "No WNU ↔ Subclone of U"

**The worry.** Post's 1941 classification lists all clones on {0,1} and their generators,
but does not use the language of "WNU polymorphism" (introduced decades later). The
claim "C has no WNU iff C ⊆ U" is an assertion about how the two characterisations
align. It is plausible but would need explicit verification by going through Post's
lattice and checking which clones contain WNU operations.

**Mitigation.** This is well-established in the CSP literature. Schaefer's 1978 theorem
explicitly identifies the six tractable classes, and the algebraic reformulation
(Jeavons et al. 1997, Bulatov et al. 2003) shows that the six tractable classes
correspond exactly to the six classes of clones containing one of the six "standard"
WNU-like polymorphisms (AND, OR, majority, affinity, const-0, const-1). Any Boolean
clone outside these six must be a subclone of U. The statement is standard.

**Residual risk.** Low. However, for a rigorous proof in the paper, one should
cite Post (1941) + Böhler–Creignou–Reith–Vollmer (2003–2004) explicitly for the
equivalence, rather than relying on it as "well-known."

### 5.2 The Constant Functions May Not Be in the Clone

**The worry.** Clone_k(L) in the programme's notation may be defined as the set of
k-ary *idempotent* polymorphisms (those satisfying f(x,…,x) = x), in which case the
constant functions const₀ and const₁ are NOT in the clone (since const₀(1,…,1) = 0 ≠ 1).
If the polymorphism clone is taken to be idempotent, then the count drops from 2k+2
to 2k (projections + negated projections), and the bound becomes |Clone_k(L)| ≤ 2k.

**Mitigation.** For most NP-complete Boolean constraint languages (including 3-SAT),
const₀ and const₁ are NOT polymorphisms of the language — 3-SAT is not preserved by
the constant-0 function (setting all variables to 0 can falsify a clause like
x ∨ y ∨ z when restricted to relations containing (1,1,1)). So for 3-SAT specifically,
Clone_k(3-SAT) ⊆ {πᵢ, ¬πᵢ} and |Clone_k| ≤ 2k.

**Resolution.** The bound 4k is safe in all cases (idempotent or not, constants in
clone or not). The tight count depends on whether constants are polymorphisms of the
specific L. The theorem as stated with ≤ 4k is safe regardless.

### 5.3 The Bridge Programme's Definition of "Clone_k(L)" May Differ from Standard

**The worry.** The Clone Growth ↔ Fullness Bridge Theorem uses a specific definition
of Clone_k(L) tied to the construction of the type III₁ factor M_Bool^L. If Clone_k(L)
in this construction means "k-ary automorphisms of the GNS Hilbert space" rather than
"k-ary polymorphisms of L as a set-theoretic relation," then the count 2k+2 might not
apply. The operator-algebraic lift could add or subtract operations relative to the
purely combinatorial count.

**Mitigation.** UA2 is explicitly labelled in the blackboard as "Direct from BZ" and
as contributing the combinatorial half of the bridge (nodes UA1, UA2 versus OA1, OA2).
The definition of Clone_k(L) relevant to UA2 is the standard polymorphism-clone
definition. The operator-algebraic interpretation is handled separately in OA1.

**Residual risk.** Medium. The node OA1 (the bottleneck) must establish that the
combinatorial clone injects faithfully into the automorphism group of M_Bool^L. If the
injection is not injective, then a linear count of combinatorial operations could map
to even fewer automorphisms, which would *strengthen* the linear growth bound on the
OA side — so this risk is conservative.

## Status: **ADVANCED**

UA2 is complete. The bound |Clone_k(L)| ≤ 4k (tight: 2k+2) follows directly from:
1. Bulatov–Zhuk: no Taylor polymorphism ↔ NP-complete;
2. Post's lattice: on {0,1}, no Taylor polymorphism ↔ clone ⊆ U (essentially unary);
3. Direct enumeration: exactly 2k+2 essentially unary k-ary Boolean functions exist.

No new mathematics is required. The node feeds directly into the Assembly (1.5)
as the "linear side" of the Clone Growth dichotomy.

The only remaining uncertainty is definitional (Section 5.2–5.3) and is conservative:
the bound 4k cannot be made worse by the concerns raised.
