### 1.2 Dissection of the garbled one-liner

## Part II: The Boolean Partition Function (Attack 8 Repair)

### 2.1 The problem

KEY LEMMA 3.4.3 asserts that the Boolean BC system has a unique KMS_1 state whose GNS factor is type III_1. The proof in Appendix B invokes a partition function Z^Bool(beta) and claims convergence properties analogous to the Riemann zeta function. The adversarial pass (Attack 8a) identified a substantive gap: the function count F(s) ~ s/log(s) cited in the proof is attributed to "Hopcroft-Karp (1973)" which is a bipartite matching algorithm, not a circuit-counting result. The actual distribution of Boolean function minimum circuit sizes is a deep open problem in circuit complexity.

### 2.2 What is known about circuit counting

The relevant question is: how many Boolean functions have minimum circuit size exactly s?

**Definition.** For each positive integer s, let

    F_n(s) = #{f : {0,1}^n -> {0,1} : C(f) = s}

where C(f) is the minimum circuit size of f (over any fixed complete basis, e.g., {AND, OR, NOT}).

**What Shannon-Lupanov tells us (the INVERSE problem):**

- Shannon (1949): Almost all Boolean functions on n variables require circuits of size Theta(2^n / n). Precisely: the fraction of n-variable functions with C(f) <= (1 - epsilon) * 2^n / n tends to 0 as n -> infinity.

- Lupanov (1958): Every Boolean function on n variables can be computed by a circuit of size (1 + o(1)) * 2^n / n.

Combined: for fixed n, the minimum circuit sizes of n-variable functions are concentrated near 2^n / n. The "spectrum" of circuit sizes is peaked near this value.

**What Shannon's counting argument gives (the FORWARD problem):**

The number of circuits of size at most s (over n inputs, with gates from a fixed complete basis) is at most

    N(n, s) <= 2^{O(s log s)}

This is proved by encoding each circuit as a string: each of the s gates requires O(log s) bits to specify its input wires (chosen from n inputs + s-1 preceding gates) and gate type. The total encoding length is O(s log s) bits, giving at most 2^{O(s log s)} distinct circuits.

Since distinct circuits can compute the same function, the number of FUNCTIONS computable by circuits of size at most s satisfies:

    #{f : C(f) <= s} <= N(n, s) <= 2^{O(s log s)}

**The cumulative counting function:**

Define G_n(s) = #{f : {0,1}^n -> {0,1} : C(f) <= s}. Then:

- G_n(s) <= 2^{O(s log s)} for all s (Shannon counting bound)
- G_n(2^n/n) = 2^{2^n} (Lupanov: all functions are covered by this size)
- G_n(s) is monotonically non-decreasing in s

The differential counting function is F_n(s) = G_n(s) - G_n(s-1).

### 2.3 The Boolean BC partition function

The Boolean BC partition function is defined as:

    Z^Bool(beta) = SUM_{f in B^P} (size_min f)^{-beta}

where B^P is the set of polynomial-time-computable Boolean functions (across all arities n) and size_min f is the minimum circuit size of f.

**Critical structural difference from the original BC system:**

In the original Bost-Connes system, the partition function is:

    Z(beta) = SUM_{n=1}^{infty} n^{-beta} = zeta(beta)

This works because there is exactly ONE positive integer of each "size" n. The counting function is F(n) = 1 for all n. The Riemann zeta function arises from this trivial counting.

In the Boolean system, there are MANY functions of each circuit size s. The partition function becomes:

    Z^Bool(beta) = SUM_{s=1}^{infty} F(s) * s^{-beta}

where F(s) = #{f in B^P : size_min(f) = s} counts the number of polynomial-time functions with minimum circuit size exactly s.

The analytic behavior of Z^Bool(beta) depends entirely on the growth rate of F(s).

### 2.4 What can we prove about F(s)?

**Upper bound on the cumulative function.** For polynomial-time functions with inputs of arity at most n, the minimum circuit sizes are at most n^c for some constant c (by definition of polynomial time). Summing over all arities:

    F(s) <= #{circuits of size s on any number of inputs}
         <= SUM_{n=1}^{s} [circuits of size s on n inputs]
         <= SUM_{n=1}^{s} 2^{O(s log s)}
         <= s * 2^{O(s log s)}

This gives F(s) <= 2^{O(s log s)}, which grows SUPER-POLYNOMIALLY.

**Lower bound on F(s).** For each n, the number of n-variable functions with circuit size in [s, s+1) is at least 1 for any s in the range [n, 2^n/n] (since circuit sizes take all integer values in this range, by a padding argument). Summing over n <= s:

    F(s) >= #{values of n such that s <= n^c for some polynomial-time function}

For polynomial-time functions specifically, we restrict to s <= n^c. For a given s, the relevant arities are n >= s^{1/c}. At each such arity, there are functions of size exactly s. The number of such functions is bounded below by the number of n-variable functions whose complexity falls in a unit interval around s, which for most s is a positive quantity.

A more careful analysis: fix n and consider the n-variable functions in P (computable by circuits of size at most n^c). The total number of such functions is at most 2^{O(n^c log(n^c))} = 2^{O(n^c * c * log n)}. These functions have circuit sizes distributed over [1, n^c]. By Shannon's counting, for small s (s << 2^n/n), the number of functions with C(f) = s is INCREASING in s (more complex functions are more numerous, up to the peak at 2^n/n). For the polynomial-time functions (s <= n^c, which is much smaller than 2^n/n for large n), we are in the left tail of the distribution where F_n(s) is increasing.

### 2.5 The analytic structure of Z^Bool(beta)

**Case 1: F(s) grows polynomially.** If F(s) ~ c * s^alpha for some alpha >= 0, then:

    Z^Bool(beta) ~ c * SUM_{s=1}^{infty} s^{alpha - beta} = c * zeta(beta - alpha)

This converges for beta > alpha + 1 and has a simple pole at beta = alpha + 1. The KMS structure would be analogous to BC 1995, shifted by alpha. The critical temperature would be beta_c = alpha + 1.

**Case 2: F(s) grows super-polynomially.** If F(s) ~ 2^{s^delta} for some delta > 0 (sub-exponential), or F(s) ~ 2^{O(s log s)} (the Shannon upper bound), then Z^Bool(beta) diverges for all real beta, because s^{-beta} * 2^{s^delta} -> infinity. The partition function would need regularization, and the standard BC KMS analysis does not directly apply.

**Which case holds?** This is the key question, and it depends on how we define the Boolean BC system.

### 2.6 Resolution: the correct definition of the Boolean BC partition function

The issue is that Appendix B conflates two different objects:

**(A) The unrestricted partition function** summing over ALL Boolean functions (across all arities), weighted by circuit size. This gives F(s) ~ 2^{O(s log s)}, super-polynomial growth, and the zeta-like structure fails.

**(B) The arity-restricted partition function.** For the Bost-Connes construction, the natural object is not a sum over all functions but a product over "primes" or a sum arising from the SEMIGROUP structure of PCirc^+.

In the original BC system, the partition function arises from the modular flow:

    sigma_t(mu_n) = n^{it} * mu_n

The partition function Z(beta) = SUM_n n^{-beta} counts semigroup elements (positive integers) weighted by their modular weight.

In the Boolean BC system, the modular flow is:

    sigma_t^Bool(mu_C) = (size C)^{it} * mu_C

The partition function should count CIRCUITS (semigroup elements of PCirc^+), not functions:

    Z^Bool(beta) = SUM_{C in PCirc^+/~} (size C)^{-beta}

where the sum is over equivalence classes of circuits under functional equivalence (two circuits are equivalent if they compute the same function).

**But this IS the function-based partition function** Z^Bool(beta) = SUM_f (size_min f)^{-beta} with multiplicities. The multiplicity of size s is F(s), the number of distinct functions with minimum circuit size s.

### 2.7 The structural resolution: stratification by arity

The correct approach, which Appendix B should adopt, is to STRATIFY by input arity n:

    Z^Bool(beta) = PROD_{n=1}^{infty} Z_n(beta)

where Z_n(beta) is the contribution from n-variable circuits. For fixed n:

    Z_n(beta) = SUM_{f : {0,1}^n -> {0,1}, f in P} (C(f))^{-beta}

The number of n-variable polynomial-time functions is at most 2^{O(n^c * log n)} (bounded by the number of circuits of polynomial size on n inputs). Their circuit sizes range from 1 to n^c.

For fixed n, Z_n(beta) is a FINITE sum (finitely many n-variable P-time functions), so convergence is not an issue. The question is whether the PRODUCT over n converges.

**Product convergence.** Write Z_n(beta) = 1 + R_n(beta) where R_n(beta) accounts for non-trivial n-variable functions. Then:

    log Z^Bool(beta) = SUM_{n=1}^{infty} log(1 + R_n(beta))

For beta large enough, R_n(beta) is small (circuit sizes >= 1, so each term is at most 1, and the number of terms at arity n grows sub-doubly-exponentially), and the sum converges.

The analytic structure (location of poles, KMS phase transition) depends on the fine structure of R_n(beta) as n -> infinity. This is where the deep circuit-complexity question enters.

### 2.8 A viable path to KMS_1 existence and type III_1

Rather than proving sharp asymptotics for F(s) -- which would likely require resolving open problems in circuit complexity -- the following approach sidesteps the partition function gap.

**Step 1: KMS_1 existence by compactness.** The space of states on A^Bool_BC is weak-* compact. The set of KMS_beta states for beta > 1 is non-empty (construct Gibbs-type states on finite-dimensional truncations). Any weak-* accumulation point as beta -> 1^+ is a KMS_1 state. This gives EXISTENCE without computing Z^Bool(beta) explicitly.

**Step 2: Faithfulness from ergodicity.** The KMS_1 state is faithful on the phase algebra C*((Z/2)^infty) because the Bernoulli(1/2) measure gives positive mass to every cylinder set, and the KMS condition forces the state to agree with the Bernoulli measure on the phase algebra (this is the standard argument from BC 1995, Section 25).

**Step 3: Type III_1 from multiplicative independence.** The Connes spectrum S(M) = R_+^* follows if the modular weights {log(size C) : C in PCirc^+} generate a dense subgroup of R. Since circuits of size 2 and size 3 both exist (elementary gates), and log 2 and log 3 are rationally independent, the subgroup generated by {log 2, log 3} is already dense in R. Therefore S(M_Bool) = R_+^*, confirming type III_1.

**Step 4: KMS_1 uniqueness.** This is the most delicate step and the one most affected by the partition function gap. In the original BC system, uniqueness of KMS_1 follows from the fact that zeta(beta) has a simple pole at beta = 1 (Bost-Connes 1995, Theorem 25). For the Boolean system, uniqueness could be established by:

(a) Showing the partition function has at most polynomial divergence as beta -> 1^+, which would follow from polynomial growth of F(s). OR

(b) Using a direct operator-algebraic uniqueness argument (e.g., the Ruelle-Perron-Frobenius theorem for the groupoid, if applicable). OR

(c) Showing that the KMS_1 state, whatever its uniqueness status, always produces a type III_1 factor with the required properties (fullness/non-fullness depending on L).

**Assessment of the three options:**

Option (a) requires resolving the F(s) growth question. For P-time functions restricted to arity n, the total number is at most 2^{O(n^c log n)}, distributed over circuit sizes in [1, n^c]. The average density per unit size interval is at most 2^{O(n^c log n)} / n^c, which is still super-polynomial. Option (a) likely fails unless a different parametrization is used.

Option (b) is viable but requires careful verification of the groupoid structure (this is essentially what the original BC 1995 does, using the left-regular representation of the ax+b group and Hecke algebra theory).

Option (c) is the most robust: even if KMS_1 is not unique, the proof chain for Parts (i) and (ii) may work for ANY KMS_1 state, not just a specific one. Path B requires a faithful normal state on M_Bool^L to define the u-topology on Aut(M_Bool^L), but the Marrakchi criterion (fullness <==> Inn closed) is independent of the choice of faithful normal state. Route C uses the crossed-product structure, which is independent of the KMS state entirely.

### 2.10 The F(s) ~ s/log(s) claim: verdict

The claim in Appendix B that F(s) ~ s/log(s) is **unsupported and likely incorrect**. The cited reference (Hopcroft-Karp 1973) is about bipartite matching, not circuit counting.

The actual growth of F(s) depends on the parametrization:

- If we fix arity n and vary s: the distribution of circuit sizes among n-variable functions is concentrated near 2^n/n (Shannon-Lupanov), with a density that depends on the "circuit complexity spectrum" -- an object that is studied but not fully characterized.

- If we sum over all arities: F(s) = SUM_n F_n(s), where F_n(s) counts n-variable functions with C(f) = s. For small s (much less than 2^n/n), the main contribution comes from arities n where s is a "typical" complexity, i.e., n ~ s * log s / log 2 (inverting the Shannon bound). The density of functions near the typical complexity is estimated as 2^{2^n} / (2^n/n) ~ n * 2^{2^n - n}, which for n ~ s log s is a tower-exponential quantity -- far from s/log(s).

**The claim F(s) ~ s/log(s) is wrong by many orders of magnitude.** The actual situation is either:

- F(s) grows super-polynomially (if summing over arities), making the partition function divergent.
- F(s) is a finite-arity quantity that should be treated per-arity, not as a global sum.

The correct resolution is the stratification approach of Section 2.7, or the state-existence-without-partition-function approach of Section 2.8.
