## 1. PATH B (Part i): Taylor implies non-full via pigeonhole

The Author claims Path B independently covers Part (i), making Gap Alpha irrelevant. The argument chain is:

```
Taylor --> exponential clone growth (UA1)
  --> exponentially many clone unitaries in U(|Sol|)
    --> pigeonhole --> close pairs
      --> Ad(u_n) --> id with u_n non-scalar
        --> Inn(M) not closed
          --> non-full by Marrakchi
```

### 1(a). Do polymorphisms actually produce UNITARIES in U(|Sol|)?

**Finding: WEAKENED.**

Polymorphisms are operations f : {0,1}^k --> {0,1} that preserve relations coordinate-wise. They are NOT unitary matrices. The lift from polymorphism to unitary goes through the "clone operator" construction in Strategy 09 (Section 3):

V_f = (1/|S|^{(k-1)/2}) * sum_{a in S} sum_{a^(2),...,a^(k) in S} |f(a,a^(2),...,a^(k))> <a|

This is a PARTIAL ISOMETRY, not a unitary. The Author then takes the polar decomposition V_f = W_f |V_f| and extends W_f to a unitary on ell^2(S). This construction has two flagged gaps:

- **[ASSUMPTION A2]**: V_f lies in M (the von Neumann algebra). The Author acknowledges this as MEDIUM severity. Strategy 12 claims A2 is "CLOSED" based on computational verification (100% for P-time), but computation on small instances is not a proof. The membership question requires V_f to be in the ultraweak closure of the algebra generated by the Hecke operators. For the polar part, the Author correctly notes that if V_f is in M then the polar unitary is automatically in M (functional calculus for von Neumann algebras). But the IF is not proved.

- **[GAP G2]**: The extension of the partial isometry to a unitary in M. As the Author notes, this is automatic IF A2 holds.

**Verdict on 1(a):** The polymorphism-to-unitary lift is a non-trivial construction requiring A2 (membership in M). The Author is honest about this but calls it "CLOSED" based on computational evidence at n = 6, 7, 8 (Strategy 12). This overstates the case. A2 is PLAUSIBLE, not CLOSED. The computational closure means "passes all tests at small n," not "proved." A rigorous proof requires checking that V_f commutes with the commutant M' (or equivalently, that V_f is expressible as a strong-limit of finite sums of generators of M). This is a standard but non-trivial verification for crossed-product algebras.

**Severity: LOW-MEDIUM.** The construction is natural and the membership question is of a standard type. Likely closable with ~2 pages of formal argument.

### 1(b). Does pigeonhole on a compact group produce approximately inner automorphisms?

**Finding: VERIFIED (with a caveat).**

The pigeonhole argument is sound in its structure: exponentially many unitaries (c * lambda^k for lambda > 1) in a compact group U(|Sol|) of FIXED dimension |Sol|^2 forces, by volume comparison, the existence of pairs (u_i, u_j) with d(u_i, u_j) < epsilon(k) where epsilon(k) --> 0 as k --> infinity. This is standard.

The step from "close unitaries" to "close automorphisms" is also correct: ||Ad(u) - Ad(v)|| <= 2||u - v|| for any unitaries u, v. So close unitaries give close inner automorphisms.

Setting w_k = u_i * u_j^{-1}, we get Ad(w_k) = Ad(u_i) * Ad(u_j)^{-1}. If u_i and u_j are close, then w_k is close to the identity, so Ad(w_k) is close to Ad(I) = id. This gives Ad(w_k) --> id.

**The caveat:** The convergence rate depends on the dimension |Sol| being FIXED as k increases. But |Sol| depends on the INSTANCE Gamma, not the language L. The Author's argument works for a FIXED instance Gamma. For the language-level statement, one needs: for each k, there exists an instance Gamma(k) such that the argument works. Since |Sol(Gamma)| can be taken to be any value >= 2 (by choosing appropriate instances), the dimension is indeed controllable. The Author addresses this implicitly (Strategy 12, section 6.2: "|Sol| >= 2 for the argument to work").

**But this introduces an instance-dependence that needs care.** The sequence u_k is constructed using different instances for different k. The limiting statement "Ad(u_n) --> id" must be interpreted in the language-level von Neumann algebra M_Bool^L, not in a single instance. This is consistent with the BC construction (M_Bool^L is the inductive limit over all instances), but the convergence needs to be verified in the inductive-limit topology. The Author does not address this point.

**Severity: LOW.** The instance-to-language-level transition is a standard feature of inductive-limit constructions, and the convergence in the inductive-limit topology should follow from the convergence at each finite level.

### 1(c). Is u_n non-scalar?

**Finding: VERIFIED.**

The Author's argument is: U(1) * I (scalar unitaries) is a 1-dimensional submanifold of U(|Sol|), which has dimension |Sol|^2. For |Sol| >= 2, the codimension is >= 3. Two generically chosen polymorphism unitaries, even if close, will not differ by a scalar multiple because the scalar multiples form a set of measure zero in U(|Sol|).

More precisely: if u_i and u_j are both polymorphism-induced unitaries from DISTINCT polymorphisms f_i != f_j, then the Author argues (Strategy 09, A4) that V_{f_i} != V_{f_j} as operators (because distinct polymorphisms disagree on some input with positive omega-measure), hence U_{f_i} != lambda * U_{f_j} for any scalar lambda (because the disagreement is on a specific matrix element, not a global phase).

The computational verification (Strategy 12, metric #5) confirms u is non-scalar for both P-time and NPC classes at k = 4.

**Severity: NEGLIGIBLE.** This step is solid.

### 1(d). Does "Inn(M) not closed" follow from Ad(u_n) --> id with u_n non-scalar?

**Finding: WEAKENED -- this is more subtle than presented.**

The Marrakchi criterion (arXiv:1605.09613) states: M is full iff for every sequence (u_n) in U(M) with Ad(u_n) --> id (in the u-topology on Aut(M)), we have u_n --> U(Z(M)).

The contrapositive: M is NOT full iff there EXISTS a sequence u_n in U(M) with Ad(u_n) --> id but u_n NOT converging to U(Z(M)).

The Author's argument provides: for each k, a unitary u_k with Ad(u_k) approximately close to id (with error epsilon(k) --> 0) and u_k non-scalar. But "u_k non-scalar" is NOT the same as "u_k does not converge to a scalar." The Marrakchi criterion requires that the SEQUENCE (u_n) does not converge to U(Z(M)) = T * 1.

**The gap:** It is possible that each u_k is non-scalar, yet the sequence u_k converges to SOME scalar lambda * 1. If the polymorphism unitaries cluster near the scalar submanifold as k increases (even though each individual u_k is non-scalar), the Marrakchi criterion would give fullness, not non-fullness.

**Does this happen?** The Author must show that the non-scalarity is UNIFORM: there exists delta > 0 such that d(u_k, T * 1) >= delta for all k. The pigeonhole argument gives u_k = u_i * u_j^{-1} where u_i, u_j are close. Even if both u_i and u_j are far from scalars, their ratio u_i * u_j^{-1} could still converge to a scalar.

**However:** The Author's construction in Strategy 09 produces unitaries that lie in the DIAGONAL algebra D. Strategy 09 Section 2.3 shows that diagonal unitaries that approximately commute with everything MUST be approximately scalar. This means the Path B construction CANNOT use diagonal unitaries alone -- it MUST use the off-diagonal construction (Section 3 of Strategy 09). The off-diagonal construction (partial isometries from the k-tuple map) produces unitaries that are NOT diagonal, and these can potentially satisfy Ad(u_k) --> id while remaining non-scalar.

The computational evidence (Strategy 12, metric #4: Ad(u) distance 0.744 at k=4) shows the sequence is NOT YET close to id at k=4. The convergence as k --> infinity is expected (section 6.1 of Strategy 12) but not proved. Moreover, the non-scalarity of the SEQUENCE (not just individual terms) is not established.

**Severity: MEDIUM.** This is a genuine analytical gap. The Author needs to prove: (i) Ad(u_k) --> id as k --> infinity (convergence rate from the pigeonhole volume estimate), AND (ii) d(u_k, T * 1) stays bounded away from 0 (uniform non-scalarity). Claim (i) follows from the standard volume argument. Claim (ii) is the harder part and requires understanding the geometry of the polymorphism-unitary family in U(|Sol|). It is PLAUSIBLE (polymorphism unitaries from distinct high-arity operations should not cluster near scalars) but not proved.

### 1(e). Does "Inn(M) not closed" imply "non-full" via Marrakchi?

**Finding: VERIFIED.**

The Marrakchi criterion (full iff Inn(M) is closed) is correctly stated and correctly cited. The direction used (Inn not closed --> non-full) is the easy direction of the biconditional and is the standard definition of fullness in the type III context.

The related HM-FULLNESS equivalence chain:
- Full <--> Inn(M) closed <--> spectral gap for sigma_t <--> no non-trivial central sequences

is correctly stated. The direction "Inn not closed --> non-full" is the contrapositive of "full --> Inn closed."

**Severity: NONE.** This step is a direct citation of an established theorem.

### Summary verdict on Path B: WEAKENED

Path B is a sound conceptual argument with two gaps:
- A2 (membership of clone operators in M): PLAUSIBLE, likely closable, severity LOW-MEDIUM.
- Uniform non-scalarity of the pigeonhole sequence: GENUINE GAP, severity MEDIUM.

The A2 gap is likely closable with formal work. The uniform non-scalarity gap is more subtle and may require additional structural analysis of the polymorphism-unitary family. Neither gap is a conceptual killer.

**Path B verdict: WEAKENED (from "confirmed" to "sound architecture with two formalization gaps").**

## 2. ROUTE 1: Houdayer-Isono for Part (ii) (Non-Taylor --> Full)

The argument chain:
```
Non-Taylor L --> G_L non-amenable --> G_L bi-exact
  --> G_L acts strongly ergodically --> no central sequences --> full
```

### 2(a). Non-Taylor --> G_L non-amenable

**Finding: WEAKENED -- the most serious gap in the entire programme.**

The Author's argument: "NP-completeness + Q_STRUCT-RESOLVED." Q_STRUCT-RESOLVED (Node 1.3.1) shows that G_Bool (the FULL circuit group acting on all of {0,1}^infty) is non-amenable because it contains Thompson's V. The Author then argues G_L is non-amenable because:

(i) NP-completeness of L means every P-time computation can be reduced to L.
(ii) The reduction circuits are elements of G_L.
(iii) G_L therefore contains enough structure to be non-amenable (specifically, contains F_2).

**The problem:** G_Bool is non-amenable, and G_L is a SUBGROUP of G_Bool (the L-preserving circuits). A subgroup of a non-amenable group CAN be amenable. The integers Z are a subgroup of F_2, and Z is amenable. So "G_Bool non-amenable" does NOT imply "G_L non-amenable."

The Author attempts to address this via Lemma 2.2.2 (F_2 inside G_L from NP-completeness). The argument is:

- NP-completeness of L means we can simulate any P-time computation as an L-instance.
- The group of reversible P-time Turing machines on {0,1}^n contains all Toffoli gates.
- Toffoli gates on >= 3 bits generate A_{2^n}.
- A_{2^n} for large n contains F_2.
- The restriction to L-preserving circuits does not eliminate the Toffoli group because "every Toffoli gate can be encoded as an L-preserving operation by the NP-completeness of L."

**Critical gap in Lemma 2.2.2:** The claim "every Toffoli gate can be encoded as an L-preserving operation" conflates two things:

(1) NP-completeness means any Boolean computation can be REDUCED to an L-SAT instance. This means there is a polynomial-time transformation that converts an arbitrary computation into an L-instance whose satisfiability encodes the computation's output.

(2) The resulting transformation is an element of G_L ONLY if it acts as an AUTOMORPHISM of the L-sector of the BC algebra. The reduction from a Toffoli gate computation to an L-instance is a map between DIFFERENT objects (a circuit computation and a constraint satisfaction instance). It is NOT an L-preserving bijection of {0,1}^n.

The Author partially acknowledges this (Section 2.1, "Remark on preserves" and the Hecke interpretation) by adopting the broader definition: G_L is the group generated by Hecke isometries mu_C and mu_C* restricted to the L-sector. Under this interpretation, G_L includes "reduction circuits" that are NOT necessarily L-preserving bijections.

**But this creates a tension:** If G_L is generated by Hecke isometries (which are not automorphisms but endomorphisms of the BC algebra), then G_L is not obviously a group of automorphisms of A|_L. The Houdayer-Isono theorem requires G acting by AUTOMORPHISMS on A. If G_L acts by endomorphisms that are only dilated to automorphisms via the Hecke construction, then the crossed-product identification (CP-1) becomes load-bearing, and the entire Route 1 depends on CP-1.

**The Author's concrete construction of F_2 inside G_{3-SAT} (Section 2.2)** is also problematic. The two proposed generators (alpha: negate x_1 and flip signs; beta: cyclic permutation of x_1, x_2, x_3) are described as acting on the CLAUSE STRUCTURE as well as the assignment string. But modifying the clause structure means these are NOT maps within a single L-instance -- they transform one instance to another. For the Hecke action interpretation, this is acceptable, but for the group-acting-on-A|_L interpretation, it requires the broader Hecke framework.

The "better construction via ping-pong" (Lemma 2.2.2) relies on embedding Toffoli gates via NP-completeness reductions. This is the key claim, and it needs a precise argument showing that the encoding of a Toffoli gate as an L-preserving operation produces an INVERTIBLE element of G_L (not just a one-way reduction).

**Severity: HIGH.** This is the most serious gap. The non-amenability of G_L for a specific NPC language L is the foundation of Route 1. The argument as written relies on an informal claim about NP-completeness reductions giving elements of G_L, but the precise mechanism (Hecke action? direct L-preserving bijections? reduction circuits?) is not nailed down. Different interpretations lead to different groups, and the Houdayer-Isono theorem needs a specific type of group action.

### 2(b). G_L bi-exact: Haagerup implies bi-exact?

**Finding: WEAKENED -- the cited implication requires careful verification.**

The Author claims: "Haagerup property implies bi-exact (Ozawa 2004, Theorem 15(iii))."

Checking this claim against the literature:

Ozawa's 2004 paper "Solid von Neumann algebras" (Acta Math. 192) contains the following:

- **Theorem 15 lists examples of bi-exact groups**, including: (i) free groups (ii) Gromov-hyperbolic groups (iii) groups acting properly on CAT(0) cube complexes.

The claim that (iii) implies "Haagerup implies bi-exact" goes through an intermediate step: a group with the Haagerup property admits a proper affine isometric action on a Hilbert space, and groups acting properly on FINITE-DIMENSIONAL CAT(0) cube complexes have the Haagerup property (Niblo-Roller). But the reverse implication -- Haagerup implies proper action on a CAT(0) cube complex -- is FALSE in general. There exist groups with the Haagerup property that do not act properly on any CAT(0) cube complex.

**However, for Thompson's V specifically:** Farley (2003) proved that V acts properly on a FINITE-DIMENSIONAL CAT(0) cube complex. So for V, bi-exactness follows from Ozawa's Theorem 15(iii) directly, NOT through the Haagerup property as an intermediate step. The logical chain is:

V acts properly on finite-dim CAT(0) cube complex (Farley 2003)
  --> V is bi-exact (Ozawa 2004, Theorem 15(iii))

This is correct. The Author's statement "Haagerup implies bi-exact" is an overstatement of the general principle, but the SPECIFIC application to Thompson's V is valid because V has the stronger property (proper action on a CAT(0) cube complex, not just Haagerup).

**The subgroup inheritance:** Ozawa's Proposition 14 states that bi-exactness passes to subgroups. This is correct and standard. IF G_L is a subgroup of V (conditional on the embedding from Q_STRUCT Claim 5.1), then G_L is bi-exact.

**The V-embedding question (Q_STRUCT Claim 5.1):** Does G_Bool (or G_L) actually embed in Thompson's V? The argument in Q_STRUCT is:

- Thompson's V is the group of piecewise-linear dyadic bijections of the Cantor set {0,1}^infty.
- Each element of V permutes finitely many subtrees and is computable by a circuit of depth O(log n).
- PCirc^+ (polynomial-time circuits) contains these as a subset.

This shows V embeds in G_Bool (V is a subgroup of G_Bool), not the reverse. For bi-exactness of G_L, we need G_L to be a subgroup of something bi-exact. V is bi-exact and V sits INSIDE G_Bool. But G_L also sits inside G_Bool. There is no reason G_L must sit inside V.

**This is a critical logical error.** The Author writes: "G_L, as a subgroup of Thompson's V" -- but G_L is NOT claimed to be a subgroup of V. Rather, V is a subgroup of G_Bool, and G_L is a different subgroup of G_Bool. The two subgroups may have a trivial intersection, a non-trivial intersection, or one may contain the other, but none of these is established.

For bi-exactness, we need EITHER:
(a) G_L is a subgroup of V (NOT established), or
(b) G_L is bi-exact by a direct argument (e.g., G_L itself acts on a CAT(0) cube complex), or
(c) G_L is exact and the Houdayer-Isono theorem can work with exactness instead of bi-exactness (it cannot -- bi-exactness is strictly needed).

**Severity: HIGH.** The bi-exactness claim for G_L has a logical gap: the subgroup relationship goes the wrong way. V embeds in G_Bool, but G_L does not obviously embed in V. The Author needs either to show G_L is a subgroup of V (unlikely -- G_L contains reductions specific to L that need not be piecewise-linear dyadic), or to establish bi-exactness of G_L by another route.

**Possible repair:** If G_L is generated by a finite set of elements, each of which lies in V (e.g., the Toffoli gates encoded as dyadic piecewise-linear maps), then G_L would be a subgroup of V. But the Hecke interpretation of G_L includes non-dyadic, non-piecewise-linear elements (arbitrary polynomial-time circuits), which may NOT lie in V.

### 2(c). Strong ergodicity: trivial amenable radical?

**Finding: WEAKENED.**

The argument for strong ergodicity proceeds:
- G_L is non-amenable (from 2(a) -- itself weakened).
- G_L has trivial amenable radical (Claim 2.4.1).
- The action is essentially free (SE-1).
- Jones-Schmidt (1987) gives strong ergodicity.

**The trivial amenable radical argument (Claim 2.4.1):** The Author's initial argument (N intersect F_2 must be trivial because F_2 has trivial amenable normal subgroups) is acknowledged as "needing refinement." The refined argument uses:

- F_2 is C*-simple (Powers 1975; BKKO 2017).
- C*-simple groups have trivial amenable radical (BKKO Corollary 1.4).

But this gives the trivial amenable radical of F_2, not of G_L. The amenable radical of G_L could be a non-trivial normal amenable subgroup that intersects F_2 trivially. The Author then pivots to Le Boudec (2022): "groups acting faithfully on the Cantor set have trivial amenable radical if the action is sufficiently non-elementary."

**Problem:** The Le Boudec result requires specific conditions on the action (the action must be "micro-supported" or of a specific type -- not just faithful and transitive on finite levels). The Author cites "Grigorchuk 2000; Nekrashevych 2005; Le Boudec 2022" without specifying which theorem from which paper applies. This is a vague citation. Grigorchuk's groups are famous examples of groups acting on rooted trees that are AMENABLE (the Grigorchuk group is amenable!), which shows that faithful transitive actions on the Cantor set do NOT automatically give trivial amenable radical.

The claim "G_L has trivial amenable radical" is PLAUSIBLE (because G_L contains F_2 and acts non-elementarily), but the proof as given is INCOMPLETE. The specific theorem from Le Boudec that applies to G_L needs to be identified and its hypotheses verified.

**Severity: MEDIUM.** The claim is likely true but the proof is not rigorous.

### 2(d). Essential freeness (SE-1)

**Finding: WEAKENED.**

The Author claims: "the stabilizer of a random point in {0,1}^infty under the Hecke action is trivial, because a specific infinite binary string is fixed by only the identity circuit in generic position."

**Problem:** The Hecke action is NOT the same as the circuit action on {0,1}^infty. The Hecke action involves AVERAGING over preimages (the mu_C and mu_C* operators). The stabilizer of a point x in {0,1}^infty under the Hecke action depends on the algebraic structure of the Hecke operators, not just on whether a circuit fixes x.

For the CIRCUIT action (each g in G_L acts as a bijection of {0,1}^infty), essential freeness means: for each non-identity g in G_L, the fixed-point set {x : g(x) = x} has measure zero. For polynomial-time circuits that compute non-trivial functions, this is plausible (a non-trivial circuit must change at least one bit for most inputs), but "most" needs to be made precise with respect to the KMS measure.

**Severity: LOW-MEDIUM.** Essential freeness is plausible and should be provable, but the argument as given conflates the Hecke action with the circuit action.

### 2(e). Crossed-product identification (CP-1)

**Finding: WEAKENED -- this is a structural gap, not a technicality.**

The Author claims M_Bool^L = A|_L rtimes G_L, where A|_L is the abelian base and G_L is the group of L-preserving circuits. This identification is modeled on the original Bost-Connes system, where M_BC = l^infty(Z-hat) rtimes Q_+*.

**The problem:** The Boolean BC system is a SEMIGROUP crossed product (by PCirc^+, a non-abelian semigroup), not a group crossed product. The original BC system uses the abelian semigroup N* (positive integers under multiplication), and the Hecke algebra dilation (Bost-Connes 1995, Section 4) produces the group crossed product by inverting the semigroup to get Q_+*. This dilation works because N* has a simple group completion (Q_+*).

For the Boolean BC system, the semigroup PCirc^+ consists of polynomial-time circuits (including non-invertible ones). The "group completion" of PCirc^+ is NOT well-defined in the same way: a non-invertible circuit cannot be formally inverted by adding a "formal inverse" in the same way that N* is completed to Q_+*. The Hecke algebra construction uses the mu_C and mu_C* operators, where mu_C* is the ADJOINT of mu_C (not the inverse circuit). The algebra generated by mu_C and mu_C* for all C is the Hecke algebra, and its GNS completion at the KMS state gives M_Bool.

**Is this GNS completion isomorphic to a group crossed product?** In the original BC system, yes (because the mu_n* operators implement the "inverse" of the multiplication-by-n endomorphism). For the Boolean system, the analogous question is whether the mu_C* operators implement "inverses" of the circuit endomorphisms in a way that produces a group action.

The Author correctly identifies this as a gap (CP-1) but rates it at 0.80 probability and calls it "TECHNICAL." I disagree with this assessment. The non-abelian, non-invertible nature of PCirc^+ makes the dilation significantly more complex than in the original BC system. This is not a routine generalization.

**Severity: MEDIUM-HIGH.** The crossed-product identification is foundational for Route 1. If M_Bool^L is NOT a group crossed product, then Houdayer-Isono does not apply directly. The Author would need either (a) to prove the dilation, or (b) to find a generalization of Houdayer-Isono to semigroup crossed products (no such generalization is known).

## 3. The "No New Mathematics" Claim

**Finding: BROKEN.**

The Author claims (Section 9, "The honest picture"): "The chain has NO steps that require fundamentally new mathematics -- only new APPLICATIONS of existing operator algebra and group theory to a new setting."

This claim is false. The following steps require genuinely new arguments that go beyond applying known theorems:

**(i) Lemma 2.2.2 (F_2 inside G_L).** The claim that NP-completeness of L guarantees a free subgroup in G_L is not an application of any known theorem. It requires a new construction (encoding Toffoli gates as L-preserving operations) that has never appeared in the literature. The argument as written is a SKETCH, not a citation of a known result.

**(ii) The V-embedding for G_L.** Claiming G_L is a subgroup of Thompson's V is a new structural claim about the Boolean BC system. No prior work has studied whether L-preserving polynomial-time circuits embed into Thompson's V. As argued in section 2(b) above, this claim appears to have the subgroup relationship backwards.

**(iii) The trivial amenable radical of G_L.** The application of Le Boudec (2022) to G_L requires verifying hypotheses (micro-support, non-elementary action) that have never been checked for G_L.

**(iv) The crossed-product identification (CP-1).** The Hecke dilation from semigroup to group crossed product in the non-abelian Boolean setting is a genuinely new result, not a routine generalization of Bost-Connes.

**(v) Essential freeness (SE-1).** While plausible, this requires a new argument about the fixed-point structure of polynomial-time circuits under the KMS measure.

Each of these is a non-trivial adaptation that amounts to new mathematics. The Author is applying known TEMPLATES (Houdayer-Isono, Jones-Schmidt, Ozawa) but the HYPOTHESES need to be verified in a new setting, and this verification is itself new mathematics.

**Verdict: BROKEN.** The "no new mathematics" claim is dishonest in the sense that it understates the difficulty of verifying the hypotheses. The templates are known; the hypothesis-verification is new.

## 5. The Combined Argument (Section 10)

### Circularity check
