# ORA Node 1.2 — UA2: Clone-Size Upper Bound for Non-Taylor Boolean Languages

## Direction

Prove that if a Boolean constraint language L on domain {0,1} does not admit a Taylor polymorphism, then the k-ary slice of its polymorphism clone satisfies |Clone_k(L)| <= 2k + 2, which is O(k) (linear growth). This tightens the 4k + 2 bound suggested in the programme brief.

---

## Research (6-Step Loop)

### Step 1 — DIAGNOSE: Identify the Structural Classification

**Theorem (BZ-DICHOTOMY, Boolean case).** [Bulatov, FOCS 2017; Zhuk, JACM 2020; for the Boolean specialization see also Schaefer 1978 and Jeavons-Cohen-Gyssens 1997.]

Let L be a constraint language over {0,1}. Exactly one of the following holds:

(a) L admits a Taylor polymorphism (equivalently, for the Boolean domain, a WNU or Mal'cev-type operation), and CSP(L) is in P.

(b) L does not admit a Taylor polymorphism, and CSP(L) is NP-complete.

In case (b), the polymorphism clone Pol(L) consists entirely of **essentially unary** operations. This is the Boolean-domain instance of the general algebraic dichotomy: on {0,1}, the only clone that fails to contain any Taylor operation is a subclone of the clone of essentially unary operations. The maximal such clone is the full clone of essentially unary operations on {0,1}.

**Reference for the Boolean specialization:** Post's lattice classifies all clones on {0,1}. The clones that contain no Taylor operation are exactly those contained in the clone traditionally denoted **I_2** (the clone of all essentially unary operations) in Post's classification. See [Post 1941], [Lau 2006, "Function Algebras on Finite Sets", Ch. 3–4].

### Step 2 — REINTERPRET: Define Essentially Unary Operations Precisely

**Definition.** A function f: {0,1}^k → {0,1} is *essentially unary* if there exists a unary function g: {0,1} → {0,1} and an index i in {1,...,k} such that for all (x_1,...,x_k) in {0,1}^k:

  f(x_1,...,x_k) = g(x_i),

or f is a constant function (depends on no variable at all). Equivalently, f depends on at most one of its k arguments.

**Definition.** A function f depends on argument i if there exist tuples a, b in {0,1}^k differing only in coordinate i such that f(a) != f(b).

### Step 3 — UNIFY: Enumerate All Unary Functions on {0,1}

The set of all functions g: {0,1} → {0,1} is:

| Name | g(0) | g(1) | Description |
|---|---|---|---|
| const-0 | 0 | 0 | constant zero |
| const-1 | 1 | 1 | constant one |
| id | 0 | 1 | identity |
| neg | 1 | 0 | negation |

There are exactly 2^2 = 4 such functions. Two are constant, two are non-constant.

### Step 4 — LOCK + COMPUTE: Count Distinct Essentially Unary k-ary Operations

We partition the essentially unary k-ary operations by how many variables they depend on:

**Case A: Functions depending on 0 variables (constant functions).**
These are f(x_1,...,x_k) = c for all inputs, where c ∈ {0,1}. Exactly **2** such functions.

**Case B: Functions depending on exactly 1 variable.**
For each coordinate i ∈ {1,...,k}, the functions that depend on exactly variable i are obtained by composing with a *non-constant* unary function g. The non-constant unary functions are id and neg. So:
- f(x_1,...,x_k) = x_i (identity on coordinate i)
- f(x_1,...,x_k) = 1 − x_i (negation of coordinate i)

For a fixed i, these are exactly 2 distinct functions. For distinct i ≠ j, x_i and x_j are distinct as k-ary operations. So we get exactly **2k** distinct functions depending on exactly one variable.

**Caution on double-counting.** f(x_1,...,x_k) = const-0(x_i) = 0 for all i is a *constant* function; it does not depend on variable i. It belongs to Case A, not Case B.

**Total count:**

|Clone_k(essentially unary)| = 2 (constants) + 2k (non-constant) = **2k + 2**.

### Step 5 — Establish the Upper Bound

**Lemma (UA2).** Let L be a constraint language over {0,1} that does not admit a Taylor polymorphism. Then for every k ≥ 1:

|Clone_k(L)| ≤ 2k + 2.

**Proof.** By BZ-DICHOTOMY restricted to the Boolean domain (equivalently, by Post's lattice classification), Pol(L) is a subclone of I_2 (all essentially unary operations on {0,1}). Therefore Clone_k(L) ⊆ {all essentially unary k-ary operations on {0,1}}. By the enumeration above, this set has cardinality exactly 2k + 2. Hence |Clone_k(L)| ≤ 2k + 2. **QED.**

**Corollary.** The growth function k → |Clone_k(L)| is O(k), i.e., at most linear.

**Remark.** The bound 2k + 2 is tight. It is achieved when Pol(L) = I_2 (e.g., L = {NAE_3}).

### Step 6 — VERIFY: Comparison with Brief

The programme brief suggested |Clone_k(L)| ≤ 4k. The over-count came from assigning all 4 unary functions to each coordinate without deducting constants. The correct tight bound is **2k + 2**. The asymptotic conclusion O(k) is unchanged.

## Self-Suspicion

1. **Is "non-Taylor Boolean clone ⊆ I_2" actually correct?** YES. Standard consequence of Post's lattice. The only clones on {0,1} not containing any Taylor operation are subclones of I_2. See [Lau 2006] or [Bodnarcuk et al. 1969].

2. **Could there be essentially unary operations I missed?** NO. The enumeration is exhaustive: 2 constants + 2k non-constant coordinate-dependent. The count 2k+2 is tight and achievable.

3. **Backward verification:** The BZ-DICHOTOMY reference (Bulatov FOCS 2017, Zhuk JACM 2020) logically entails that non-Taylor Boolean clones consist of essentially unary operations. The Post lattice classification (Post 1941) independently confirms this for the Boolean case. Both sources directly entail the conclusion, not merely fail to contradict it.
