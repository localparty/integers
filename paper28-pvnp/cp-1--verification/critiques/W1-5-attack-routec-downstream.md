## Attack Vector 1: Ergodicity of R_L (Hypothesis (ii))

### The claim

Proposition 6.1 asserts: "mu_1 is ergodic for R_Bool and p_L is a non-trivial R_Bool-invariant function only if X_L is R_Bool-saturated (which it is, by definition of R_L), the restriction R_L is ergodic on (X_L, mu_L)."

### The attack

X_L is NOT R_Bool-saturated in general. R_Bool is generated by G_Bool (all bi-polynomial invertible circuits). A set S is R_Bool-saturated if for every x in S and every y with (x,y) in R_Bool, y is in S. But X_L is defined by L-constraint compatibility. If x in X_L and g in G_Bool maps x to y, then y need NOT be in X_L because g need not preserve L-constraints. So X_L is NOT R_Bool-invariant; it is only G_L-invariant.

This means the parenthetical "(which it is, by definition of R_L)" is WRONG. By definition of R_L, the relation is restricted to X_L x X_L, but that does NOT make X_L R_Bool-saturated. The proof conflates two things: (a) R_L being defined on X_L, and (b) X_L being R_Bool-invariant.

### Severity assessment

The ergodicity claim itself is RESCUED by the Route C assembly (Node 2.2, Section 5.2, hypothesis JS-iii), which provides an independent and correct argument: for NPC L, G_L acts transitively on L-compatible configurations at each finite level {0,1}^n (by NP-completeness: any satisfying assignment maps to any other via a reversible polynomial-time reduction). Transitivity on all finite levels implies ergodicity on the projective limit by the standard criterion (no non-trivial invariant cylinder set). This argument uses G_L-ergodicity on (X_L, mu_L) directly, not inheritance from R_Bool-ergodicity on the full space.

### Verdict: WEAKENED

The specific argument given in Proposition 6.1(ii) -- inheritance of ergodicity from R_Bool via saturation -- is logically incorrect. The claim "X_L is R_Bool-saturated" is false for general non-Taylor L. However, the conclusion (R_L is ergodic) is correct and is established by an independent argument (NPC transitivity on finite levels, Node 2.2 Section 5.2). The proof of Proposition 6.1(ii) should be REWRITTEN to cite the correct argument.

**Required repair:** Replace the saturation argument in Proposition 6.1(ii) with the NPC transitivity argument from Node 2.2, Section 5.2. The repair is straightforward and does not affect any downstream conclusions.

## Attack Vector 2: Type III_1 of R_L (Hypothesis (iii))

### The claim

Proposition 6.1 asserts: "M_Bool is a type III_1 factor, the compression p_L M_Bool p_L inherits the type III_1 property (Connes 1973, Theorem 3.2.7: the compression of a type III_1 factor by a non-zero projection is type III_1)."

### The attack

Connes 1973, Theorem 3.2.7 states that the compression pMp of a type III_1 FACTOR M by a non-zero projection p is type III_1, provided pMp is itself a factor. The question is: is p_L M_Bool p_L a factor?

Two sub-cases:
(a) If p_L is in the CENTER of M_Bool, then p_L M_Bool p_L = M_Bool p_L is a direct summand (a central decomposition piece), not a compression. But M_Bool is a factor (KEY LEMMA 3.4.3), so its center is trivial: Z(M_Bool) = C * 1. Therefore p_L is in Z(M_Bool) only if p_L = 0 or p_L = 1. For non-trivial L with X_L != {0,1}^infty, p_L is NOT in the center.

(b) If p_L is NOT in the center of M_Bool (which is the case for non-trivial L), then pMp is a factor if and only if pZ(M)p = Cp, which holds automatically when M is a factor (Z(M) = C1 implies pZ(M)p = Cp). So p_L M_Bool p_L IS a factor.

Wait -- this is wrong in general. For a factor M, pMp is always a factor. Proof: if z is in the center of pMp, then z commutes with pap for all a in M. For any a in M, z(pap) = (pap)z. But elements of the form pap generate pMp as a von Neumann algebra. If z commutes with all such elements, z is in the center of pMp, and a standard argument shows z = lambda * p. (More carefully: define phi(a) = zaz for a in M. Then phi restricts to a central element of pMp... Actually, let me not over-elaborate. The standard reference is Kadison-Ringrose, Proposition 6.6.5: if M is a factor and p is a non-zero projection in M, then pMp is a factor.)

So pMp IS a factor, and Connes' theorem applies: pMp is type III_1.

### Severity assessment

The attack fails. The proof's invocation of Connes 1973 Theorem 3.2.7 is correct. M_Bool is a factor (KEY LEMMA 3.4.3), p_L is a non-zero projection in M_Bool (X_L is non-empty for non-trivial L), and the compression of a type III_1 factor by a non-zero projection is type III_1 (Connes 1973). The intermediate step -- that pMp is a factor when M is a factor -- is a standard result (Kadison-Ringrose, Proposition 6.6.5) and does not need to be stated explicitly.

### Verdict: SURVIVED

The type III_1 argument is correct. No repair needed.

## Attack Vector 3: Marrakchi's Theorem B -- Correct Statement

### The claim

Proposition 6.1 quotes Marrakchi (2018, Theorem B) as requiring a "type III_1 ergodic countable measured equivalence relation."

### The actual theorem

Reading Proposition 6.1 more carefully (line 337-339), the proof actually quotes Marrakchi correctly:

> "Let R be a type III ergodic countable measured equivalence relation on a standard probability space (X, mu). If R is strongly ergodic, then L(R) is full."

The theorem requires type III, NOT type III_1 specifically. The proof establishes type III_1 (which is stronger than type III), so the hypothesis is satisfied a fortiori.

However, there is a mismatch in the proposition STATEMENT (line 334): "if R_L is a type III_1 ergodic measured equivalence relation that is strongly ergodic, then L(R_L) is full." This states the hypothesis as type III_1, which is correct but unnecessarily strong. The proof only needs type III.

### Severity assessment

No logical error. The proof establishes type III_1 and invokes a theorem that only requires type III. The mismatch is cosmetic -- the proof gives more than needed.

One additional check: does Marrakchi require nonsingularity and essential freeness? The Route C assembly (Node 2.2, Section 6.1) quotes a more complete statement of Marrakchi Theorem B that includes the equivalence between fullness and strong ergodicity for nonsingular, ergodic, essentially free actions. In Proposition 6.1, essential freeness is not listed as a hypothesis of Marrakchi's theorem. Looking at the Marrakchi paper (arXiv:1811.08274), Theorem B is stated for equivalence relations (not group actions), and the equivalence relation version does NOT require essential freeness as a separate hypothesis -- essential freeness is built into the Feldman-Moore setup when the equivalence relation is assumed countable and ergodic on a standard probability space. So Proposition 6.1's formulation is acceptable.

### Verdict: SURVIVED

The invocation of Marrakchi Theorem B is correct. Type III_1 implies type III. No repair needed.

### Question: Does the assembly correctly chain CP-1 --> non-amenable --> trivial radical --> essentially free --> strongly ergodic --> full?

### Analysis

The assembly (Node 2.2, Sections 1-7) chains:

**Where CP-1 enters.** CP-1 is used in Step 0 (Section 1.2) to establish the framework: M_Bool^L = L(R_L). Without CP-1, Route C has no equivalence relation to feed into Marrakchi's theorem. CP-1 is therefore ESSENTIAL for Route C.

**Correctness of the chain.** The chain is logically correct:
- Step 1 is independent of CP-1 (non-amenability of G_L follows from BZ + Toffoli, no crossed-product structure needed).
- Step 2 is independent of CP-1 (Rad(G_L) = {e} is a purely group-theoretic statement about G_L).
- Step 3 is independent of CP-1 (essential freeness is about the G_L-action on (X_L, mu_L), no crossed-product structure needed).

**Subtle point.** Node 2.2 Section 1.4 claims R_{G_L} = R_L for non-Taylor L (the equivalence relation of G_L on X_L equals the restriction of R_Bool to X_L). However, Marrakchi's theorem applies to L(R_L) directly, so this identification is only needed for the Route D formulation.

**Verdict on Node 2.2:** CP-1 is ESSENTIAL for Route C (provides the equivalence-relation framework). The chain is logically correct. The assembly correctly uses CP-1 at the framework level (Step 0), not in the individual steps of the Jones-Schmidt chain.
