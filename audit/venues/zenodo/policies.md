# Zenodo Policies

**Source:** https://about.zenodo.org/policies/ + https://help.zenodo.org/
**Retrieved:** 2026-04-14

## Eligible depositors
> Anyone may register as user of Zenodo. All users are allowed to deposit content for which they possess the appropriate rights.

## DOI allocation
- Content receives a DOI upon deposit.
- If violations are found, the Zenodo DOI is invalidated and the record updated to indicate the original external DOI.

## Withdrawal policy
- Withdrawn items are replaced with tombstone pages explaining the reason.
- The original DOI and URL remain active (preserves citation integrity).
- Withdrawals require justification from uploader or reasonable contact attempts.

## Content restrictions
- Must not violate privacy, copyright, or confidentiality.
- Accepts research data from any lifecycle stage in all fields.
- "All fields of research. All types of research artifacts."

## File size limits
- Standard quota: 50 GB per record.
- Larger limits negotiable case-by-case.

## Version control
- Data files are versioned.
- Records are NOT versioned — each version gets a unique DOI but the "concept DOI" is stable.
- Original content is preserved; derivatives may be generated.

## Metadata
- JSON schema standards.
- License required for public files.
- English preferred for textual items (all languages accepted).
- Exports: MARCXML, Dublin Core, DataCite.

## GitHub integration
- GitHub releases can be automatically deposited as Zenodo records.
- Each release gets a DOI; a "concept DOI" represents the software.
- Setup via https://zenodo.org/account/settings/github/.

## Implications for QG5D programme
- **Zenodo is the FIRST step in the publication cascade** (strategy/ym §9).
- Establishes priority timestamp.
- DOI is citable.
- Does NOT qualify as a Clay Qualifying Outlet (not refereed, not MathSciNet-indexed).
- Perfect for: bare + full deliverables, data bundles, code repos, supplementary materials.
