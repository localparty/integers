# research/03-dir5 -- 36 Predictions as Finite Rigidity Check

*Date: 2026-04-10. Spectral Realisation Cycle 3, Direction 5.*

## 1. The question

Can the 36 zero-parameter predictions rule out extra eigenvalues?
Each prediction f_i(gamma_{n_1}, ...) = m_i (measured). If an extra
eigenvalue lambda existed, would it perturb any prediction beyond
experimental error?

## 2. The structure of the 36 formulas

The 36 predictions fall into two classes:

### Class 1: Individual matrix elements (33 formulas)

These use SPECIFIC gamma_n values:
- CC hierarchy: exp(gamma_1 * pi^2/2)
- t_0: (log gamma_7)^2 Gyr
- n_s: gamma_9 / gamma_10
- H_0: gamma_11 * 4/pi
- m_t: gamma_3 * gamma_8 / (2*pi)
- m_W: gamma_2 + gamma_13
- etc.

Each formula is <n|L_hat|n> or a function thereof. These are
matrix elements of L_hat in the eigenbasis. An extra eigenvalue
|lambda> adds a new state but does NOT change <n|L_hat|n> for
existing n. The eigenvalues gamma_n are INTRINSIC to the operator;
adding a state doesn't shift them (self-adjoint, no level repulsion
from extra dimensions since L_hat is diagonal in its own basis).

### Class 2: Trace-type formulas (0 of the 36)

None of the 36 predictions involve sums over ALL eigenvalues.
The Laurent coefficients a and b use gamma_E and zeta(2), which
are derived from the zeta function at s=1, but the formulas
themselves are closed-form constants, not sums over {gamma_n}.

## 4. The trace constraint (Direction 5 repurposed)

If we move beyond the 36 individual predictions and consider TRACE
identities:

### The spectral zeta function:

    zeta_T(s) = sum_n gamma_n^{-s}

If there's an extra eigenvalue lambda:

    delta zeta_T(s) = lambda^{-s}

At s = 2:

    lambda = 15: delta = 0.00444 (28% of zeta_T(2) ~ 0.016)
    lambda = 17: delta = 0.00346 (22%)
    lambda = 20: delta = 0.00250 (16%)

These are LARGE fractional perturbations. But there is no
independent computation of zeta_T(2) from the BC side.

### The resolvent:

    Tr((T_BC - z)^{-1}) = -zeta'/zeta(z)  (if compact resolvent)

Extra eigenvalue at lambda:

    delta Tr = 1/(z - lambda)

At z = 14.57 (midpoint to gamma_1 = 14.13):
    lambda = 15: delta = -2.31  (HUGE)
    lambda = 17.5: delta = -0.59
    lambda = 20: delta = -0.34

These are enormous -- but the identity Tr = -zeta'/zeta requires
compact resolvent, which is the thing we're trying to prove.

## 5. The Riemann-von Mangoldt counting constraint

N(T) = (T/2pi) log(T/2pie) + 7/8 + S(T)

With the Trudgian bound |S(T)| < 0.137 log T + 0.443 log log T + 4.350:

**The Weyl error is too large to exclude even a single extra
eigenvalue at ANY height.** This is because the error term S(T)
is O(log T), which dominates the effect of a single extra
eigenvalue.

## 6. The key structural observation

The 36 predictions DON'T use trace formulas. They use individual
eigenvalues. An extra eigenvalue:
- Does NOT change any individual gamma_n
- Does NOT enter any of the 36 formulas
- IS invisible to the 36-prediction check

The anti-fine-tuning bound P < 10^{-34} measures the probability
that the 36 formulas ACCIDENTALLY match experiment if the operator
is randomly chosen. It does NOT measure the probability of extra
eigenvalues, because extra eigenvalues don't affect the 36 formulas.

## 7. Can the argument be strengthened?

### 7.1 Use trace formulas as additional predictions

If we ADD new predictions of the form:

    sum_{gamma_n <= T} f(gamma_n) = F(T)  [explicit formula]

where F(T) is computed from the prime side (via the explicit
formula), then an extra eigenvalue WOULD perturb the left side
while the right side is fixed.

But: the explicit formula is UNCONDITIONAL. It already includes
ALL zeros, including potential off-line zeros. The identity:

    sum_rho h(gamma_rho) = [prime side]

is true for ALL zeros rho of zeta, not just those on the critical
line. An extra eigenvalue in spec(T_BC) that is NOT a zero of zeta
would violate this identity.

This IS a constraint: spec(T_BC) can only contain zeros of zeta.
But this is exactly Meyer's spectral inclusion (already known):
{zeros of zeta} subset spec(T_BC). The reverse inclusion requires
compact resolvent.

### 7.2 Use the operator dictionary completeness

If the operator dictionary is EXHAUSTIVE -- every matrix element
of L_hat is accounted for by some formula -- then an extra state
|lambda> would need to appear somewhere. If it appears nowhere,
it can't exist.

Problem: the dictionary has finitely many entries (36). The matrix
of L_hat is infinite-dimensional. Even if 36 diagonal elements
<n|L_hat|n> are accounted for, there are infinitely many others.
The dictionary is not exhaustive.

## 8. Premise check

**Does this argument distinguish spec = {gamma_n} from
spec = {gamma_n} union {extra}?**

NO. The 36 predictions are identically zero-perturbed by extra
eigenvalues. The counting constraint (Weyl) is too loose. The
trace constraint requires compact resolvent (circular).

**Direction 5 is VACUOUS for proving spectral realisation.**

The anti-fine-tuning bound remains the best EVIDENCE: the operator
has the right eigenvalues for 36 independent measurements, which
is astronomically unlikely by chance. But "unlikely" is not
"impossible," and the perturbation from extra eigenvalues is
exactly zero, so even infinite precision wouldn't help.

## 9. Verdict

**DEAD END for a proof.** The 36 predictions cannot detect extra
eigenvalues because they use individual matrix elements, not
traces. This is a STRUCTURAL limitation, not a numerical one:
no finite set of individual-eigenvalue formulas can exclude
extra eigenvalues.

**Value as evidence:** Still enormous (P < 10^{-34}). But the
gap between evidence and proof is unbridge-able along this
direction.

**Corrective to the original hypothesis:** The hypothesis
("each prediction's error budget is too tight for extra
eigenvalues") was WRONG because the predictions don't depend
on extra eigenvalues at all. The perturbation is exactly zero,
not "small."
