## 2. Convergence Analysis

### 2.1 Model Fits

Three models were fit to the data sigma(n) for n = 12, 14, 16, 18, 20:

**Model A: sigma = sigma_inf + c/n^2** (YM finite-size correction)
- sigma_inf = 0.00755
- c = -0.825
- R^2 = 0.771

**Model B: sigma = a * n^(-alpha)** (power-law decay to zero)
- a = 0.0041
- alpha = 0.010 (essentially constant)
- R^2 = -0.008 (terrible fit -- the data is NOT consistent with sigma -> 0)

**Model C: sigma = sigma_inf + c/n^beta** (general finite-size scaling)
- sigma_inf = 0.00557
- c = -629.3
- beta = 4.81
- R^2 = 0.811

### 2.2 Interpretation

Model B (power-law decay to zero) has NEGATIVE R^2, meaning it fits worse than a constant. The data categorically rejects sigma -> 0.

Models A and C both predict sigma_inf > 0:
- Model A: sigma_inf = 0.0076 +/- 0.001
- Model C: sigma_inf = 0.0056 +/- 0.002

**Best estimate: sigma_inf = 0.006 +/- 0.002**

The finite-size scaling exponent beta = 4.8 (Model C) indicates that the correction terms die off very rapidly. By n=16, the finite-size effects are already subdominant.

### 2.3 Why the Prior Two-Point Extrapolation Was Misleading

The A5 data had only n=12 and n=14, both in the regime where sigma is INCREASING due to the constraint graph becoming dense enough to support the area law. The slight decrease from n=12 to n=14 (sigma = 0.00170 -> 0.00132) was a statistical fluctuation, not a trend. At the same alpha=3.5 used here, the n=14 value is 0.00283, which is higher than the A5 n=14 value (the difference is likely due to different alpha values: A5 may have used a different clause ratio).

The current data with five n-values clearly shows the non-monotonic behavior: increase from n=12 to n=16, then stabilization at ~0.005.

## 4. Self-Suspicion

### 4.1 The Horn-SAT Noise Floor

**Concern:** Horn-SAT shows sigma ~ 0.001, not exactly 0. This sets a systematic noise floor.

**Assessment:** The noise floor is a sampling artifact. At n >= 14, Horn-SAT instances have thousands of solutions and the WalkSAT sampler cannot find them all. Majority of a subset can fail even when majority is a true polymorphism. The 2-SAT control (which has fewer solutions at alpha=1.0 and can be fully enumerated) shows sigma = 0.000 exactly. The 3-SAT signal at sigma ~ 0.005 is 5x above the Horn-SAT noise floor, providing a clean separation.

**Mitigation:** Future work could use a uniform solution sampler (e.g., UniGen based on approximate model counting) instead of WalkSAT to reduce the sampling noise for P-time classes.

### 4.2 Large Variance in sigma

**Concern:** The standard deviation of sigma within each n is comparable to the mean (coefficient of variation ~ 1). Individual instances vary substantially.

**Assessment:** This is expected for random 3-SAT instances near the satisfiability threshold. The AVERAGE sigma is positive and stable across n, and the MEDIAN is also positive (except at n=14 where 4 out of 15 instances had sigma=0 due to very few solutions). The instance-level variance does not affect the language-level conclusion: on average, 3-SAT has positive string tension.

### 4.3 The Non-Monotonic Behavior

**Concern:** sigma increases from n=12 to n=16 before stabilizing. This is surprising if sigma is a monotonically decreasing finite-size quantity.

**Assessment:** The non-monotonicity is explained by the constraint graph density. At small n with clause ratio alpha=3.5, the constraint graph is sparse and short cycles are rare. As n grows, the graph becomes denser and the area law becomes better resolved. The true string tension is only visible when the constraint graph is dense enough to support cycles of all lengths (L=3 to 8). At n=12, the constraint graph with 42 clauses on 12 variables has only ~21 edges, barely enough for L=8 cycles. At n=20, the graph has 70 clauses on 20 variables with ~35 edges, giving much better cycle coverage.

This interpretation is supported by the holonomy profile: the H(L=8) values increase from 0.061 (n=14) to 0.106 (n=16) to 0.094-0.095 (n=18,20), showing the same stabilization pattern as sigma.

### 5.2 Parameters

### 5.3 Holonomy Defect Definition

H_restricted(C) = fraction of (a,b,c) triples from Sol^3 where majority(a,b,c) violates at least one constraint that touches the cycle C (clause has >= 2 variables in C).

### 5.4 String Tension Extraction

sigma = slope of linear fit H_restricted vs Area(L), using the A5-calibrated area table.
