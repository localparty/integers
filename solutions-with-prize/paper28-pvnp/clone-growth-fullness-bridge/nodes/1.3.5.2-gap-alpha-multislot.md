The multi-slot construction produces a genuine NEW phenomenon (decreasing commutators, no rank-1 collapse) that is structurally different from K-14. However, the decrease is attributed to increasing contractiveness, not increasing centrality. The distinction is:

- **Centrality:** T approximately commutes with everything: ||[T,y]|| -> 0 for all y, while T is non-trivial.
- **Contraction:** T maps everything to a low-dimensional subspace: ||T*x - c*x|| -> 0 for some fixed point, which trivially reduces commutators because the image is shrinking.

The multi-slot construction is producing contraction, not centrality. Gap Alpha remains OPEN -- neither closed nor definitively killed for multi-slot constructions.

### 2.3 Symmetrized = single-slot (exact identity)

The symmetrized construction T_f^{sym} is numerically identical to the single-slot T_f^{(1)} for threshold-k polymorphisms. This is a theorem, not a numerical coincidence:

**Proposition.** If f(x_1,...,x_k) is symmetric in all k arguments (i.e., invariant under any permutation of inputs), then T_f^{sym} = T_f^{(1)}.

*Proof.* T_f^{(j)} places the live slot at position j and averages the remaining (k-1) slots. By the symmetry of f, permuting which slot is live does not change the operator: T_f^{(j)} = T_f^{(1)} for all j. Hence T_f^{sym} = (1/k) * k * T_f^{(1)} = T_f^{(1)}.

This means: **symmetrization is useless for symmetric polymorphisms.** To get a genuinely different symmetrized operator, one must use ASYMMETRIC polymorphisms (e.g., f(x,y,z) = maj(x, y AND z, z) or similar non-symmetric compositions).

## 3. Self-Suspicion (Three Ways)

### 3.1 Suspicion 1: Is the multi-slot decrease a contraction artifact?

**Assessment: HIGH risk. This is likely the correct explanation for the decrease.**

The per-b slice T(:,:,b) is a Markov-like operator on Sol. As k increases:
- The (k-2) averaged slots concentrate onto the mean configuration
- T(:,:,b) approaches: T_b[s, a] = delta(f(a, b, mu, ..., mu), s) where mu is the omega-mean
- This is a DETERMINISTIC function of (a,b), producing a PERMUTATION MATRIX (or sub-permutation)
- A permutation matrix has SMALL commutators with matrix units (it simply permutes rows/columns)

The commutator ||[P, y]||_2 for a permutation matrix P and matrix unit y = |i><j| equals:
- 0 if P permutes i to j (exact commutation)
- O(1/sqrt(S)) otherwise

So as T_b approaches a deterministic mapping, the commutator decreases. This is NOT centrality -- it is the operator becoming MORE RIGID (less stochastic), not more central.

**Diagnostic:** If the decrease is a contraction artifact, then the NORMALIZED commutator ||[T_b, y]|| / ||T_b - tau(T_b)*I|| should NOT decrease. The ratio should stay constant or increase, because the operator is getting closer to a fixed point (smaller numerator AND smaller denominator).

### 3.2 Suspicion 2: Does the SV decline mean the multi-slot will eventually rank-collapse too?

**Assessment: MEDIUM risk.**

The SV ratio declines as ~ k^{-0.55}. This is SLOWER than the single-slot rank collapse (where the effective rank drops to 1). Extrapolation suggests the SV ratio remains positive for all finite k:
- At k=100: ratio ~ 0.10
- At k=1000: ratio ~ 0.03

The two live slots structurally prevent COMPLETE rank-1 collapse because the output f(a, b, c, ...) always depends on BOTH a and b through the polymorphism's non-trivial interaction. However, the residual 2D structure may not be sufficient for Gap Alpha -- it is a 2D subspace of an S-dimensional space, and the commutator behavior depends on whether this 2D structure is "central" or "concentrated."

**Prediction:** At very high k, the multi-slot operator approaches a rank-2 projector (projecting onto the subspace spanned by the two live slots' interaction with the omega-mean). This rank-2 projector is NOT approximately central -- it is a fixed 2D projection that does not commute with generic matrix units.

### 3.3 Suspicion 3: Is the k=11 uptick on the second instance the beginning of a reversal?

**Assessment: MEDIUM risk.**

Instance 2 (|Sol|=10) shows multi-slot commutator norms:
- k=3: 0.475, k=5: 0.377, k=7: 0.348, k=9: 0.345, k=11: 0.359

The k=9 -> k=11 change is +0.014, breaking the monotonic decrease. This could be:
(a) Monte Carlo noise (the MC budget is limited at high k)
(b) The beginning of a reversal (the contraction effect saturates and rank compression takes over)
(c) A finite-size artifact at |Sol|=10

The primary instance (|Sol|=32) does NOT show this uptick (strict monotonic decrease through k=11). The uptick may be a small-|Sol| effect. Resolving this requires data at k=13, 15, 17 and larger instances.
