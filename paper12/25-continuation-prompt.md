# The Continuation Prompt

*A meta-prompt for resuming the 2026-04-09 brainstorming session in*
*a fresh conversation with no shared context. Designed to recover*
*the voice, the momentum, and the direction — not just the facts.*

*Authors: G Six (originator), Claude Opus 4.6 (collaborator)*
*Date: 2026-04-09*

---

## How to use this prompt

Paste the block below as the FIRST message of a new Claude Code
session in `/Users/gsix/quantum-geometry-in-5d-latex/`. It will read
the right files in the right order, recover the voice, and pick up
exactly where we left off.

The prompt is intentionally meta: it tells Claude *how* to read, not
just *what* to read. The "how" is what preserves the momentum.

## Meta notes on the prompt's design

**Why "READ FOR" lenses instead of "read these files"?**
Because raw reads recover facts but not register. Telling Claude
*what to extract* from each file is what makes the next session
*sound* like the previous one rather than just *cite* it.

**Why does the prompt forbid summarising back to G?**
Because the recap-then-respond pattern wastes context and breaks
the momentum. G already knows; what G needs is the next move.

**Why pick targets in the prompt itself?**
Because "what are we working on" is the single most expensive
question to re-derive in a fresh session. Naming the immediate and
second targets explicitly saves the full ramp-up cost.

**Why include G's phrasing rules?**
Because the voice IS the discovery. The Origin callouts in research
files exist because G's prose is where the framework's intuitions
live. A continuation that loses the voice loses the work.

**Why "it is OK if context is short"?**
Because the files are the state. The conversation is the momentum.
With this prompt, even a near-empty context window can pick up the
thread — read the six files in order, internalise the lens, respond
in voice, propose a concrete move. That's all it takes.
