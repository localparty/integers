PART 1: Analytic y-integrals — closed-form expressions
======================================================================

Baseline integrals over [0, pi*R]:
  int cos(n y/R) dy = pi*R * delta(n, 0)  for all n >= 0
  int sin(n y/R) dy = R/n * (1 - (-1)^n)
                    = 2R/n  if n is odd
                    = 0     if n is even (including n=0)

NOTE: The sin integral is NONZERO for odd n. This is a half-period
domain [0, pi*R], not the full period [-pi*R, pi*R].

I_{+++}(n1,n2,n3) = int cos(n1 y/R) cos(n2 y/R) cos(n3 y/R) dy
  Expand via cos*cos = [cos(A-B)+cos(A+B)]/2:
  = (pi*R/4)[delta(n1-n2-n3,0) + delta(n1-n2+n3,0)
              + delta(n1+n2-n3,0) + delta(n1+n2+n3,0)]

  Key cases (n1,n2,n3 >= 0):
    All zero (n1=n2=n3=0):   I_{+++} = pi*R
    One zero, two equal:     I_{+++} = pi*R/2
    Diagonal n1=n2=n3=n>0:  I_{+++} = 0  (no delta fires)
    Triangle n3=n1+n2>0:    I_{+++} = pi*R/4
    All distinct, generic:  I_{+++} = 0

I_{++-}(n1,n2,n3) = int cos(n1) cos(n2) sin(n3) dy
  Reduces to sum of sin integrals:
  = (1/4)[S(n1-n2+n3) - S(n1-n2-n3) + S(n1+n2+n3) - S(n1+n2-n3)]
  where S(m) = int sin(m y/R) dy = 2R/m if m odd, 0 if m even.

  NOT always zero. Nonzero when sin arguments include odd numbers.
  RELEVANT FOR: mixed h_{mu nu}/h_{mu5} vertices (not GS sunset).

I_{+--}(n1,n2,n3) = int cos(n1) sin(n2) sin(n3) dy
  Reduces to cos integrals (sin*sin -> cos via product-to-sum):
  = (pi*R/4)[delta(n1-n2+n3,0) + delta(n1+n2-n3,0)
              - delta(n1-n2-n3,0) - delta(n1+n2+n3,0)]

  Nonzero at triangle configurations (with sign).
  RELEVANT FOR: vertices with one h_{mu nu} + two h_{mu5}.

I_{---}(n1,n2,n3) = int sin(n1) sin(n2) sin(n3) dy
  Reduces to sum of sin integrals:
  = (1/4)[S(n1-n2+n3) - S(n1-n2-n3) - S(n1+n2+n3) + S(n1+n2-n3)]
  where S(m) = 2R/m if m odd, 0 if m even.

  NOT always zero. RELEVANT FOR: all-h_{mu5} vertices (not GS sunset).

PART 3: GS sunset topology and selection rules
======================================================================

The Goroff-Sagnotti (GS) operator:
  C_GS * R_{mu nu rho sigma} R^{rho sigma lam tau} R_{lam tau}^{mu nu}
  (also written as Riem^3 or R^3_GS)

This operator involves only the Riemann tensor, built from the metric
h_{mu nu}. It does NOT involve h_{mu 5} or phi.

At two loops, the sunset diagram contributing to C_GS has:
  - Three internal graviton propagators: all h_{mu nu}^{(n)}
  - Two cubic vertices: each vertex couples three h_{mu nu} legs
  - KK numbers: lines carry levels (n, m, n+m) by KK momentum conservation

Z2 parity of the internal lines:
  h_{mu nu}^{(n)} has mode function cos(ny/R): Z2-even (+)

At each vertex: three Z2-even legs → parity product = (+)(+)(+) = +1
The vertex integral is I_{+++}(n1, n2, n3).

Parity combinations at a cubic vertex:
      Parities    Integral  parity  Status/orbit
       (+,+,+)     I_{+++}      OK  Product = +1  ALLOWED
       (+,+,-)     I_{++-}      NO  Product = -1  FORBIDDEN (orbifold symmetry)
       (+,-,-)     I_{+--}      OK  Product = +1  ALLOWED
       (-,-,-)     I_{---}      NO  Product = -1  FORBIDDEN (orbifold symmetry)

Note: I_{++-} and I_{---} are ALGEBRAICALLY nonzero (corrected earlier)
but FORBIDDEN by Z2 orbifold symmetry. The orbifold requires the action
to be invariant under y → -y; vertices must have Z2-even product.
The Z2-FORBIDDEN vertices (I_{++-}, I_{---}) vanish in the action,
not because the integral is zero, but because the orbifold projection
removes them (they correspond to Z2-odd operators).

For the GS sunset: ONLY I_{+++} contributes.

KK momentum conservation at each vertex:
  Vertex 1: lines n1, n2, -(n1+n2) with coupling I_{+++}(n1,n2,n1+n2)
  Vertex 2: lines n1, n2, -(n1+n2) with coupling I_{+++}(n1,n2,n1+n2)
  (Both vertices are identical in the sunset; the two loop momenta
   share the same KK structure.)

  1/(k^2 + m^2) = (1/k^2) [1 - m^2/k^2 + (m^2/k^2)^2 - ...]

  J(m_n^2) = J(0) + m_n^2 * J'(0) + m_n^4 * J''(0) + ...
