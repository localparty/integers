# research/03-dir6 -- Uniqueness of omega_1 -> Compute Spectral Measure

*Date: 2026-04-10. Spectral Realisation Cycle 3, Direction 6.*

## 1. The question

omega_1 is unique. The spectral measure d_mu of T_BC w.r.t. omega_1:

    omega_1(e^{itT_BC}) = integral e^{it*lambda} d_mu(lambda)

Can we compute omega_1(e^{itT_BC}) from the BC algebraic structure
INDEPENDENTLY of the zeros, and show it equals sum_n a_n e^{it*gamma_n}
(purely atomic)?

## 2. The trivial case: omega_1(e^{itT_BC})

The modular flow identification:

    sigma_t(A) = e^{itT_BC} A e^{-itT_BC}

Applied to A = 1:

    sigma_t(1) = 1  (the identity is always fixed by automorphisms)

Therefore:

    omega_1(e^{itT_BC} * 1 * e^{-itT_BC}) = omega_1(1) = 1

But this gives omega_1(1) = 1, not omega_1(e^{itT_BC}). The issue:

    omega_1(e^{itT_BC}) = <Omega_1, e^{itT_BC} Omega_1>

This IS the spectral measure of T_BC w.r.t. the cyclic vector
Omega_1. It is NOT the same as omega_1(sigma_t(1)) = omega_1(1) = 1.

So: omega_1(e^{itT_BC}) is non-trivial and DOES depend on t.

## 3. Computing omega_1(e^{itT_BC}) from the BC structure

### The BC time evolution

sigma_t(mu_n) = n^{it} * mu_n

On the GNS Hilbert space H_1, T_BC has eigenvalues {log n} with
eigenvectors {pi_1(mu_n) Omega_1} (up to normalization).

The spectral decomposition on H_1:

    Omega_1 = sum_n c_n |log n>

where |log n> is the eigenstate of T_BC with eigenvalue log n,
and c_n = <log n | Omega_1>.

The KMS_1 condition gives:

    |c_n|^2 = n^{-1} / zeta(1)

But zeta(1) = infinity. The state omega_1 is NOT a trace on A_BC
(this is the type III nature). The vector Omega_1 is NOT normalizable
in the naive sense.

### The regularized spectral measure

The spectral measure of T_BC w.r.t. Omega_1 on H_1 is:

    mu(dx) = sum_n |c_n|^2 delta(x - log n)

where |c_n|^2 ~ n^{-1} (KMS_1 weights). This gives:

    omega_1(e^{itT_BC}) = sum_n n^{-1} * n^{it} / Z
                        = sum_n n^{it-1} / Z
                        = zeta(1 - it) / zeta(1)

But zeta(1) = infinity, so this ratio is 0/infinity in a
distributional sense.

The CORRECT interpretation: omega_1(e^{itT_BC}) should be read
as a distributional statement. In the GNS representation:

    <Omega_1, e^{itT_BC} Omega_1> = lim_{beta -> 1+} zeta(beta - it) / zeta(beta)

As beta -> 1+:
- zeta(beta) ~ 1/(beta - 1)
- zeta(beta - it) ~ 1/(beta - 1 - it) for t near 0

So: the ratio ~ (beta - 1)/(beta - 1 - it) -> 1/(1 + it/(beta-1))
-> delta-function behavior as beta -> 1.

This is the DEGENERATE limit: the spectral measure concentrates
at log n = 0 (n = 1) as beta -> 1, which is the type III_1
behavior.

## 4. Non-trivial test element: A = mu_2

Instead of omega_1(e^{itT_BC}), compute:

    omega_1(mu_2 * e^{itT_BC} * mu_2*)

where mu_2 is the Hecke operator at prime 2.

sigma_t(mu_2) = 2^{it} * mu_2, so:

    omega_1(mu_2 * e^{itT_BC} * mu_2*)
    = omega_1(mu_2 * sigma_t(mu_2*))  [no, this is wrong]

Actually:

    mu_2 * e^{itT_BC} * mu_2* = mu_2 * sigma_t(mu_2*)
    [no, e^{itT_BC} acts by conjugation for sigma_t, not by left multiplication]

Let me be more careful. In the GNS representation:

    <pi_1(mu_2) Omega_1, e^{itT_BC} pi_1(mu_2) Omega_1>

The vector pi_1(mu_2) Omega_1 has expansion:

    pi_1(mu_2) Omega_1 = sum_n c_n * pi_1(mu_2) |n>

where pi_1(mu_2)|n> involves the Hecke action on the BC algebra.
For mu_2, the Hecke operator acts as:

    mu_2 f(x) = f(x/2) (for f in C(Z_hat*))

In the spectral decomposition, mu_2 maps |n> to |2n>:

    pi_1(mu_2) |n> ~ |2n>  (up to normalization)

So:

    <pi_1(mu_2) Omega_1, e^{itT_BC} pi_1(mu_2) Omega_1>
    = sum_n |c_n|^2 * e^{it * log(2n)}
    = sum_n n^{-1} * (2n)^{it} / Z
    = 2^{it} * zeta(1 - it) / zeta(1)

This is just 2^{it} times the previous expression. The phase
factor 2^{it} carries information about the prime 2, but the
same divergence issues persist.

## 5. The fundamental obstacle

The spectral measure of T_BC on H_1 is:

    mu_{H_1}(dx) = sum_{n=1}^inf n^{-1} delta(x - log n)

This is the measure with weights {n^{-1}} at positions {log n}.
It is DIVERGENT (sum n^{-1} = infinity) reflecting the type III_1
nature.

The spectral measure on H_R (if it exists) would be:

    mu_{H_R}(dx) = sum_n a_n delta(x - gamma_n)

with some weights {a_n}.

Computing mu_{H_R} from the BC algebra requires projecting from
H_1 to H_R. This projection is the CONTENT of spectral realisation.

The algebraic computation of omega_1(A e^{itT} B) gives mu_{H_1},
not mu_{H_R}. To get mu_{H_R}, we need:

    omega_R(e^{itT_BC}) = <Omega_R, e^{itT_BC} Omega_R>

where Omega_R is the cyclic vector in H_R. But constructing
Omega_R requires H_R, which requires spectral realisation.

## 6. Is there a non-circular way?

### Approach: the explicit formula as a bridge

The Weil explicit formula:

    sum_rho h(gamma_rho) = h(i/2) + h(-i/2) - sum_p sum_k ...

The left side is the spectral trace on H_R (sum over zeros).
The right side is computable from primes (BC algebra).

If we define:

    omega_R(e^{itT_BC}) := sum_rho e^{it*gamma_rho} * w_rho

then by the explicit formula with h(r) = e^{itr}:

    omega_R(e^{itT_BC}) = [prime-side expression]

This IS the independent computation. But:

(a) The explicit formula is valid for ALL zeros, including
    potential off-line zeros. The sum on the left includes
    ALL rho, not just those with Re(rho) = 1/2.

(b) The test function h(r) = e^{itr} is not in the Schwartz
    class; the explicit formula requires h to be a Schwartz
    function. So this specific computation is formal.

(c) Even with a Schwartz test function, the explicit formula
    gives sum_rho h(gamma_rho) = [prime side], which is a
    distributional statement. It doesn't tell us the spectral
    TYPE (discrete vs continuous).

## 7. The algebraic computation of the spectral measure

Suppose we compute omega_1(A e^{itT} A*) for many test elements
A, and show that for EVERY A, the function

    t -> omega_1(A e^{itT} A*)

is a sum of exponentials (purely atomic Fourier transform).

This would prove mu is purely atomic, hence pure point spectrum.

But: computing omega_1(A e^{itT} A*) from the BC algebra gives
expressions involving zeta(1-it) and its derivatives. These are
MEROMORPHIC functions of t, not sums of exponentials. The zeta
function has poles at t = -i*(1-s) for s = 1, i.e., at t = 0
(distributional). Its Fourier transform is NOT purely atomic.

**The spectral measure of T_BC on H_1 is NOT purely atomic.**
This is consistent: on H_1, T_BC has both point spectrum ({log n})
and essential spectrum (from type III_1).

The purely atomic property is hypothesized for H_R, not H_1.
And computing the spectral measure on H_R requires constructing
H_R.

## 8. Premise check

**Does this argument distinguish spec_{H_R} = {gamma_n} from
spec_{H_R} = {gamma_n} union {extra}?**

NO. The algebraic computation gives the spectral measure on H_1,
not on H_R. The H_1 spectral measure is NOT purely atomic (type
III_1). The H_R spectral measure requires constructing H_R, which
is the spectral realisation problem.

The direction is CIRCULAR: to compute the spectral measure on H_R,
you need H_R; to construct H_R, you need the spectral measure.

## 9. Verdict

**CIRCULAR.** The attempt to compute the spectral measure
algebraically runs into the same H_1 vs H_R distinction as
Direction 4. On H_1, the computation is doable but gives a
NON-atomic spectral measure (type III_1). On H_R, the computation
requires H_R to already exist.

The explicit formula provides a formal bridge between the two
spaces, but it is distributional and does not determine the
spectral type.

**The direction fails not from a wrong step but from the wrong
starting point:** the spectral measure on H_1 is not the spectral
measure on H_R, and no amount of algebraic manipulation on H_1
can determine the spectral type on H_R without first constructing H_R.
