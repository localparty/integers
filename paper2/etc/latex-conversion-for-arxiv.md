## Short abstract (for arXiv submission form, ≤ 1920 characters)

We compute the cosmological predictions of the 5D e-dimension framework
using CAMB v1.6.6, fitting zero parameters to CMB data. Two observational
inputs ($\rho_\Lambda$, $\Omega_{DM}/\Omega_b$) plus a thawing dilaton
($w_0=-0.85$, $w_a=-0.23$) determine every observable. The central result:
$\Omega_{DM}/\Omega_b = 1/\xi^2$ from bulk leptogenesis on the $Z_2$
orbifold, where $\xi = T_{hidden}/T_{visible}$. Three scenarios bracket
the prediction ($\xi = 0.432$--$0.47$): $t_0 = 13.47$--$13.60$ Gyr;
$S_8 = 0.753$--$0.785$ (resolving the $4\sigma$ clustering tension);
$H_0 = 68.7$--$69.5$ km/s/Mpc; $r_d = 145.8$--$146.2$ Mpc. The washout
correction shifts $\xi$ from 0.432 to $\approx 0.49$, converging with
the independently $\theta^*$-matched value. $H(z)$ peaks 4--6% above
$\Lambda$CDM at $z \sim 0.3$--$0.7$, testable by DESI DR3 at $8\sigma$.
$N_{eff} = 3.31$--$3.39$, in $3$--$4\sigma$ tension with ACT DR6 —
the primary open issue, resolved by CMB-S4 at $>9\sigma$.

### How arXiv handles figures

arXiv compiles your LaTeX on their servers. Key rules:

1. **Flat directory.** arXiv accepts either a flat tarball or a
   single-level directory. All figure files go **next to** the .tex
   file — no subdirectories. So rename or copy:
   ```
   etc/age/plot_summary.png  →  fig_summary.png
   etc/age/plot_s8.png       →  fig_s8.png
   etc/age/plot_Hz.png       →  fig_Hz.png
   etc/age/plot_wz.png       →  fig_wz.png
   etc/age/plot_ages.png     →  fig_ages.png
   ```

2. **Accepted formats:** PNG, PDF, EPS, JPG. PNG at 300 dpi is fine
   for raster plots (which is what matplotlib produces). PDF is
   preferred for vector diagrams but not needed here.

3. **\includegraphics without extension.** Best practice is to omit
   the file extension so LaTeX picks the right format:
   ```latex
   \begin{figure}
     \includegraphics[width=\columnwidth]{fig_summary}
     \caption{Six-panel comparison of the 5D framework predictions
       (Scenarios A, B, C) against Planck \LCDM. See text for
       parameter definitions.}
     \label{fig:summary}
   \end{figure}
   ```
   (This works because pdflatex finds .png automatically.)

4. **Two-column figures.** For wide plots (Hz, summary), use
   `figure*` environment to span both columns:
   ```latex
   \begin{figure*}
     \includegraphics[width=\textwidth]{fig_Hz}
     \caption{...}
     \label{fig:Hz}
   \end{figure*}
   ```

5. **Size limit.** Total upload ≤ 50 MB. Our five PNGs at 300 dpi
   are well under this (~2 MB total).

## arXiv submission checklist

### What to upload

Pack into a single `.tar.gz`:

```
paper2.tex          — the main LaTeX file
refs.bib            — bibliography (from paper2/etc/refs.bib)
fig_summary.png     — renamed from etc/age/plot_summary.png
fig_s8.png          — renamed from etc/age/plot_s8.png
fig_Hz.png          — renamed from etc/age/plot_Hz.png
fig_wz.png          — renamed from etc/age/plot_wz.png
fig_ages.png        — renamed from etc/age/plot_ages.png
```

No subdirectories. No .aux, .log, or .out files (arXiv generates
these). Do NOT include the Python scripts or results.json.

You have two options for the bibliography:
(a) Include `refs.bib` and let arXiv run BibTeX (simpler).
(b) Pre-build the `.bbl` file locally (`pdflatex → bibtex → pdflatex
    → pdflatex`) and include the `.bbl` instead of `.bib`. This avoids
    BibTeX errors on arXiv's servers. If you include the `.bbl`, do NOT
    also include `refs.bib`.

### Submission steps

1. Go to https://arxiv.org/submit
2. Log in (create account if needed — takes ~24h for first approval)
3. Choose **New submission**
4. Primary category: **hep-th** (High Energy Physics - Theory)
5. Cross-list: **astro-ph.CO**, **gr-qc**
6. Upload the .tar.gz
7. arXiv compiles it — check the preview PDF carefully
8. If compilation fails, check the log (common issues: missing
   packages, wrong file paths, BibTeX errors)
9. Submit — paper appears next business day at ~20:00 UTC

### Common first-time gotchas

- **BibTeX:** arXiv runs `pdflatex → bibtex → pdflatex → pdflatex`.
  If you use `\bibliography{refs}`, the file must be `refs.bib`.
  Alternatively, you can include a pre-built `.bbl` file instead.
- **\LCDM macro:** Define `\newcommand{\LCDM}{\Lambda\text{CDM}}`
  in your preamble — you'll use it dozens of times.
- **revtex4-2:** This is already installed on arXiv's TeX Live. No
  need to upload it.
- **Compilation timeout:** arXiv has a ~5 minute compile limit. Not
  an issue for this paper.
- **Abstract field:** arXiv has a separate abstract text field on the
  web form (plain text, no LaTeX math). Use `$` for inline math
  there — arXiv renders it. Keep it under the character limit.
- **Moderation hold:** First-time authors sometimes get held for
  manual review (~1-2 days). Normal — just wait.
- **Paper 1 cross-reference:** If Paper 1 is already on arXiv, cite
  it by arXiv ID: `\cite{Paper1}` with `eprint = {YYMM.NNNNN}` in
  refs.bib. If submitting both simultaneously, arXiv lets you
  cross-reference between papers in the same submission batch.
